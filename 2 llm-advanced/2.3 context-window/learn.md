# 2.3 context-window — 上下文窗口技术

## 概念定义

上下文窗口（Context Window）是 LLM 在生成下一个 Token 时能够"看到"的最大输入 Token 数量。它是 Agent 的工作记忆容量——决定了 Agent 能记住多少历史对话、工具调用结果和外部信息。

## 核心挑战

| 挑战 | 说明 |
|------|------|
| 有限容量 | 即使 128K 窗口在长时间 Agent 任务中也会耗尽 |
| 注意力稀释 | 长上下文中模型对关键信息的注意力被稀释（"大海捞针"问题） |
| 计算成本 | 注意力机制的计算量随序列长度平方增长 |
| 位置编码 | 位置编码需要支持比训练时更长的序列 |

## 技术演进

| 阶段 | 方案 | 窗口大小 |
|------|------|----------|
| 早期 | 绝对位置编码 | 2K-4K |
| 中期 | 相对位置编码（RoPE, ALiBi） | 8K-32K |
| 现在 | 窗口扩展（YaRN, NTK-aware） | 128K-1M |
| 未来 | 无限上下文（Infini-Attention, 循环） | 理论无限 |
