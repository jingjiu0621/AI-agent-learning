# 2 llm-advanced/ — 大语言模型原理与推理进阶

## 模块概述

大语言模型（LLM）是 Agent 的"大脑"——它负责理解用户意图、推理决策、生成回复和工具调用。本模块深入 LLM 的核心技术原理，包括 Tokenizer、推理机制、上下文窗口、结构化输出以及 API 设计模式。理解这些底层机制对于构建高效、可靠的 Agent 系统至关重要。

## 核心思想

1. **Tokenizer 是 LLM 的第一层"感知"**——同样的文本用不同 Tokenizer 切分结果大不相同，直接影响模型的 Token 消耗和理解能力
2. **推理机制决定性能和成本**——KV Cache、Flash Attention、量化等技术是 LLM 工程落地的关键
3. **上下文窗口是 Agent 的"工作记忆"**——如何高效利用有限的上下文是 Agent 系统设计的核心挑战
4. **结构化输出是 Agent 的基础能力**——从 JSON Mode 到 Grammar-constrained 解码，多种方案解决"LLM 输出不稳定"问题
5. **API 设计影响系统的可靠性**——流式、重试、限流、降级是生产级 LLM 应用的必备能力

## 关键工程启示

- Token 计数是 Agent 成本估算的基础——理解 Tokenizer 原理有助于准确预测和控制成本
- 推理优化（量化、缓存）对生产部署的延迟和成本有直接影响
- 上下文窗口管理（压缩、滑动、摘要）是构建长时间运行 Agent 的必要条件
- 结构化输出是从"LLM 实验"到"LLM 工程"的分水岭
