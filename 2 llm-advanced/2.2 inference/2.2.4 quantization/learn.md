# 量化推理：INT4/INT8/FP16 精度与性能权衡

## 简单介绍

量化（Quantization）是将模型权重和激活值从高精度格式（FP32, FP16）转换为低精度格式（INT8, INT4）的过程。量化可以大幅减少模型的显存占用和计算量，使大模型能在消费级 GPU 或 CPU 上运行。核心权衡在于：**精度越低，速度和显存越好，但模型输出质量可能下降**。

## 基本原理

### 数值表示

| 格式 | 位宽 | 表示范围 | 精度 | 显存节省 |
|------|------|---------|------|---------|
| FP32 | 32 bit | ±3.4×10³⁸ | 高 | 1x |
| FP16 | 16 bit | ±65,504 | 中 | 2x |
| BF16 | 16 bit | ±3.4×10³⁸ | 中（与 FP32 相同范围） | 2x |
| INT8 | 8 bit | -128 ~ 127 | 低 | 4x |
| INT4 | 4 bit | -8 ~ 7 | 极低 | 8x |

### 量化过程

```python
# 简化的量化示例
def quantize_to_int8(weights: np.ndarray) -> tuple:
    """将 FP32 权重量化为 INT8"""
    # 1. 找到权重的范围
    w_max = np.max(np.abs(weights))
    
    # 2. 计算缩放因子
    scale = w_max / 127  # INT8 范围是 [-127, 127]
    
    # 3. 量化
    quantized = np.clip(
        np.round(weights / scale), 
        -127, 127
    ).astype(np.int8)
    
    return quantized, scale

# 反量化
def dequantize(quantized: np.ndarray, scale: float) -> np.ndarray:
    return quantized.astype(np.float32) * scale
```

## 量化方案

### 1. 训练后量化（Post-Training Quantization, PTQ）

最简单、最常用的方案：

```
完整模型（FP16） → 校准数据集 → 量化（INT8/INT4） → 量化模型
```

**优点**：不需要重新训练
**缺点**：大压缩比（4bit）下质量损失明显

### 2. 量化感知训练（Quantization-Aware Training, QAT）

在训练过程中模拟量化操作，让模型"适应"量化误差：

```
训练时前向传播使用量化后的权重
反向传播使用"直通估计器"（Straight-Through Estimator）
```

**优点**：质量损失更小
**缺点**：需要重新训练

### 3. 常用量化方法

| 方法 | 精度 | 是否需要校准数据 | 特点 |
|------|------|----------------|------|
| GPTQ | 4-bit | 是 | 基于 Hessian 矩阵的权重量化 |
| AWQ | 4-bit | 是 | 基于激活值的感知量化 |
| GGUF | 2-8 bit | 否 | CPU 友好，支持灵活位宽 |
| BitsAndBytes (QLoRA) | 4-bit | 否 | 双重量化，训练推理都可用 |

## 量化对 Agent 系统的影响

```
FP16 模型: 最高质量，但需要更多显存 → 适合复杂推理、工具选择
INT8 模型: 质量略有下降，显存减半 → 适合大多数场景
INT4 模型: 质量明显下降，显存 1/4 → 适合简单任务、高并发
```

关键发现：**工具调用（Function Calling）准确性对量化非常敏感**——4-bit 量化模型的工具选择准确率可能下降 5-15%。

## 核心挑战

| 挑战 | 说明 | 缓解 |
|------|------|------|
| 离群值 | 权重的少数极大值导致量化误差大 | SmoothQuant 等特殊处理 |
| 逐层差异 | 不同层对量化的敏感度不同 | 混合精度量化 |
| 激活值量化 | 动态范围的激活值更难量化 | 动态量化 vs 静态量化 |
| 知识保留 | 大压缩比下模型可能"忘记"知识 | 结合蒸馏 |

## 工程启示

- Agent 系统中，建议对工具调用和推理质量要求高的使用 FP16 或 INT8 量化（不要低于 8-bit）
- 简单的数据处理或文本分类可以使用 INT4 量化模型
- 量化模型的"幻觉率"通常会上升——需要额外的输出验证
- 主流的推理引擎（vLLM, llama.cpp, TensorRT-LLM）都已支持量化推理
- 量化的最大好处不是"让模型变小"，而是"让相同显存可以运行更大的模型或更大的批处理"
