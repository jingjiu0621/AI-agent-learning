# 2.2 inference — 推理机制与性能优化

## 概念定义

推理（Inference）是 LLM 根据输入生成输出 Token 的过程。与训练不同，推理注重延迟、吞吐量和成本。理解推理的底层机制（自回归生成、注意力计算、KV Cache 等）是优化 Agent 性能的前提。

## 核心问题

LLM 推理面临一组矛盾：

| 目标 | 挑战 |
|------|------|
| 低延迟 | 自回归生成必须逐个 Token 生成，无法并行 |
| 高吞吐 | 注意力机制的平方级计算复杂度 |
| 低成本 | 大模型推理需要大量 GPU 显存和算力 |
| 长上下文 | 长序列下注意力计算和 KV Cache 急剧增长 |

## 推理优化的两条主线

### 1. 算法优化（软件层面）

- **KV Cache**：缓存历史 Token 的 Key/Value，避免重新计算
- **Flash Attention**：IO 感知的注意力算法，减少显存读写
- **Speculative Decoding**：用小模型"起草"大模型"验证"

### 2. 硬件优化（模型层面）

- **量化**：INT4/INT8 推理，降低显存和计算需求
- **Batch Inference**：批量处理提高 GPU 利用率
- **模型蒸馏**：小模型模仿大模型输出
