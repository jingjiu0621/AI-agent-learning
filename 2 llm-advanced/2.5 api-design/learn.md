# 2.5 api-design — LLM API 设计模式

## 概念定义

LLM API 设计模式涵盖了 Agent 系统与 LLM 服务交互时的工程实践——包括流式架构、重试策略、限流处理、批处理、模型降级和成本控制。这些模式决定了 Agent 系统在生产环境中的可靠性、响应速度和成本效益。

## 核心挑战

| 挑战 | 说明 |
|------|------|
| 高延迟 | LLM 请求通常需要数百毫秒到数秒 |
| 不确定性 | 模型可能超时、返回错误格式、或拒绝回答 |
| 成本高昂 | Token 计费模式下，低效的 API 调用会迅速增加成本 |
| 速率限制 | API 提供商限制每分钟/每月的请求频率 |
| 模型变化 | 模型版本更新可能改变行为特征 |

## 设计原则

| 原则 | 说明 |
|------|------|
| 容错优先 | 假设 API 会失败，设计优雅降级路径 |
| 流式交互 | 不要让用户等待完整的生成结果 |
| 成本感知 | 每个 API 调用都有成本，设计时考虑 Token 预算 |
| 渐进式 | 先用快模型，有需要时升级到强大模型 |
