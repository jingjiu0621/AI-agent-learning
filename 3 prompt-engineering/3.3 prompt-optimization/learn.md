# 3.3 prompt-optimization — Prompt 优化

## 本节概述

Prompt 优化是提示工程中最具工程挑战性的环节。当 Agent 系统从原型阶段进入生产阶段，Prompt 不再是"写一次就够"的静态文本，而是一个需要持续迭代、度量和改进的软件工件。本节涵盖从人工经验优化到自动优化算法、从 A/B 测试到回归测试的完整优化工具箱。

## 核心思想

1. **不可评估则不可优化**——没有量化指标，任何"优化"都是主观偏好
2. **优化是搜索问题**——Prompt 空间巨大（离散且不连续），需要结构化搜索策略
3. **局部最优陷阱**——针对单一样本过度优化会损害泛化能力
4. **退化是常态**——优化一个维度（如降低延迟）可能损害另一个维度（如准确率），需多目标平衡

## 关键工程启示

- 建立"基线 + 对照组"的评估习惯，不要相信单次跑出来的"感觉变好了"
- 优先用人工优化做方向探索，用自动优化做参数精调——前者省时间，后者省人力
- Prompt 回归测试套件应纳入 CI/CD，与代码回归测试同等对待
- 优化成果需要版本化记录，否则回退时无从下手
