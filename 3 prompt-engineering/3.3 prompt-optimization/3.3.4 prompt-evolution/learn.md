# 3.3.4 prompt-evolution — 进化算法优化 Prompt

## 简单介绍

进化算法优化 Prompt 是将遗传算法（GA）、进化策略（ES）等生物启发式搜索方法应用于 Prompt 空间。通过"变异-交叉-选择"的迭代过程，在大量 Prompt 变体中筛选出表现最优者。这种方法特别适合 Prompt 中离散参数（指令措辞、示例选择、格式设计）的联合优化。

## 基本原理

进化优化遵循达尔文式自然选择：

```python
def evolve_prompt(initial_population, eval_fn, generations=20):
    population = initial_population  # 初始 Prompt 候选集
    for gen in range(generations):
        # 评估适应度
        fitness = [eval_fn(p) for p in population]
        # 选择：保留适应度最高的个体
        selected = selection(population, fitness, top_k=5)
        # 交叉：组合两个 Prompt 产生后代
        offspring = crossover(selected)
        # 变异：随机修改 Prompt 组件
        offspring = mutate(offspring, mutation_rate=0.1)
        # 更新种群
        population = selected + offspring
    return population[0]  # 返回最优个体
```

关键的进化操作：
- **变异**：替换指令同义词、调整示例顺序、修改输出格式描述
- **交叉**：将两个 Prompt 的子结构（如角色设定和指令部分）重组
- **选择**：基于评估指标保留 top-k 个体

## 背景

进化算法在 NLP 领域的应用已有二十年历史，但直到 LLM 时代才被用于 Prompt 优化。2023-2024 年间，多个研究（EvoPrompt、PromptBreeder）证明进化策略在 Prompt 优化上能超越随机搜索和人工基线，特别是在搜索空间巨大的场景下。

## 之前针对这个问题的做法与结果

在进化方法流行前，Prompt 优化主要靠人工试错和简单的网格搜索。对于需要同时优化多个组件（指令 + 示例 + 格式 + 约束）的复杂 Prompt，人工方法几乎无法覆盖完整的组合空间。进化算法在多项 Benchmark 上取得了 5-15% 的改进，且发现了许多人类不会尝试的 Prompt 模式。

## 核心矛盾

进化优化的核心矛盾是**种群多样性与收敛速度的平衡**。多样性太低（种群同质化）会过早收敛到局部最优；多样性太高（变异率过大）会破坏已经发现的好模式，永远无法收敛。此外，进化算法的评估成本是"种群规模 x 代数"的乘积，成本压力远大于单点搜索方法。

## 当前主流优化方向

- **语义变异**：用 LLM 生成语义上合理的变异，而不是随机替换 token（如 PromptBreeder）
- **多目标进化**：同时优化准确率、成本和延迟，使用 NSGA-II 等帕累托优化算法
- **自适应超参数**：动态调整变异率和交叉率，在探索和利用之间自动平衡
- **分层进化**：先进化 Prompt 结构（角色/格式），再进化内容（措辞/示例）

## 实现的最大挑战

- **计算成本巨大**：假设种群 20 个个体进化 20 代，每代评估一次就是 400 次 LLM 调用
- **表征设计困难**：Prompt 不是固定长度的向量，如何将其编码为可交叉变异的表征是一个开放问题
- **局部最优陷阱**：进化算法容易在早期收敛到看似不错但并非最优的 Prompt
- **评估噪声**：LLM 输出的随机性使得适应度评估不准确，误导选择方向

## 能力边界与结果边界

- 进化优化适合中等规模的搜索空间（数十到数百个可调组件），但不适合极小空间（人工即可）或极大空间（计算成本不可接受）
- 通常需要 10-50 代才能收敛，取决于种群规模和变异策略
- 更适合连续优化的场景——从已有 Prompt 出发持续进化，而非从零开始搜索
- 进化发现的 Prompt 可能高度特化于训练评估集，需要额外的泛化验证

## 与其他技术的区别

- vs APE/OPRO：进化方法使用生物学启发的随机变异，APE 使用 LLM 采样，OPRO 使用 LLM 作为优化器
- vs DSPy：DSPy 的编译优化更像梯度下降（在连续空间中搜索），进化算法更适合离散空间
- vs 手工优化：进化方法可以探索人类不会考虑的变异，但找到的结果可能违反语言直觉

## 核心优势

- **无梯度**：不需要计算梯度，适用于 Prompt 这种离散不可微的优化对象
- **并行性**：种群中每个个体的评估可以完全并行，适合分布式计算
- **全局搜索**：相比局部搜索方法（如爬山法），进化算法有更好的全局探索能力
- **多目标原生支持**：可以同时优化多个互相冲突的目标

## 工程优化方向

- 使用 LLM-as-judge 作为快速适应度评估器，降低评估成本
- 实现早期停止策略——如果连续多代没有改进则终止进化
- 维护精英档案（elitism），确保最优个体不会在交叉和变异中丢失
- 开发可视化工具，展示进化轨迹——每代适应度的分布变化

## 适合场景的判断标准

- Prompt 有多个可独立变异的组件（如指令、示例、格式、约束）
- 难以凭直觉判断"哪部分应该改"——需要算法探索
- 有充足的计算预算（数百到数千次 LLM 调用）
- 场景相对稳定，进化出的 Prompt 可以长期复用
