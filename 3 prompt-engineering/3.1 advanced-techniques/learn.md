# 3.1 advanced-techniques — 高级推理提示技术

## 本节概述

高级推理提示技术是让 LLM 超越"问答匹配"、执行真正**复杂推理**的核心方法体系。从简单的思维链引导到多路径搜索与验证，这些技术构成了 Agent 系统"思考能力"的工程基础。本节覆盖 6 种关键技术，它们从不同角度解决同一个根本问题：**如何让 LLM 在不确定性和复杂性中做出可靠推理。**

## 核心思想

1. **推理显式化**——让 LLM 的"思考过程"暴露在 token 空间中，使其可观察、可干预、可优化
2. **多路径优于单路径**——单一推理链容易陷入局部最优，多路径探索 + 聚合是提高可靠性的关键模式
3. **搜索即推理**——当推理空间足够大时，将推理建模为搜索问题（树/图），用搜索算法代替直觉判断
4. **自动化的阶梯**——从手工 CoT 到自动 CoT，提示工程正在从"手工艺"向"系统工程"演进

## 关键工程启示

- **计算量与可靠性之间的权衡**——多路径技术（ToT、GoT、SC）显著提高推理质量，但代价是数倍到数十倍的 token 消耗，需要根据场景做工程取舍
- **技术与 Agent 架构的配合**——高级推理技术不是独立 Prompt 技巧，而是需要嵌入 Agent 主循环：规划阶段用 ToT/GoT，执行阶段用 CoT，验证阶段用 SC
- **评估先行**——没有标准化的推理质量评估体系，任何提示技术优化都是盲目的。在引入高级推理技术前，先建立评估基线
