# 语义工具搜索

## 简单介绍

语义工具搜索使用 Embedding 向量和相似度计算，从大量工具中检索出与用户意图最相关的工具子集。与关键词搜索不同，语义搜索能理解"发邮件"和"send_email"之间的同义关系。

## 基本原理

```
用户输入: "帮我查北京的天气"
    ↓ Embedding
[0.12, 0.87, -0.34, ...]  (查询向量)
    ↓ 余弦相似度计算
工具库向量:
  get_weather:   向量 → 相似度 0.92 ✅
  search_city:   向量 → 相似度 0.85
  send_email:    向量 → 相似度 0.23
  create_order:  向量 → 相似度 0.15
    ↓
Top-K 结果: [get_weather, search_city]
```

## 搜索策略

### 1. 直接查询搜索

直接用用户输入作为查询：

```python
query = "查北京天气"
results = vector_store.similarity_search(query, k=10)
# → get_weather, search_city, ...
```

**优点**：简单直接
**缺点**：用户输入可能很短，语义信息不足

### 2. 查询改写增强

先用 LLM 将用户输入改写为更适合搜索的形式：

```python
# 原始用户输入: "今天热死了"
# LLM 改写: "查询天气信息、气温数据"
# 用改写后的查询进行搜索
```

### 3. 多查询搜索

生成多个角度的查询进行搜索，结果融合：

```python
queries = [
    "查询天气",
    "获取气温数据",
    "城市天气预报"
]
all_results = [search(q, k=5) for q in queries]
final = fuse_results(all_results)  # 多路融合
```

### 4. 上下文增强搜索

不仅使用当前用户输入，还加入对话历史信息：

```python
context = "用户之前问了上海，现在问天气如何"
query = "上海天气"
# 结合上下文，搜索更准确
```

## 相似度计算方法

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| 余弦相似度 | 向量夹角的余弦值 | 通用 |
| 点积 | 向量内积 | 归一化向量 |
| 欧氏距离 | 向量空间距离 | 稠密向量 |
| MIP（Maximum Inner Product） | 最大内积搜索 | ANN 索引 |

## 性能优化

| 优化 | 效果 | 实现 |
|------|------|------|
| ANN 索引 | 大幅加速搜索 | HNSW / IVF |
| 量化 | 减少内存占用 | Product Quantization |
| 分片 | 并行搜索 | 按工具类别分片 |
| 缓存 | 热点查询加速 | Redis 缓存 Top-K 结果 |
| 预过滤 | 缩小搜索空间 | 先按类别再搜语义 |

## 实现挑战

1. **查询意图与工具语义的匹配**：用户说的可能是"帮我发个消息给张三"，但工具有 send_sms, send_email, send_wechat——需要区分哪种"消息"
2. **冷启动与反馈循环**：没有历史数据时搜索质量差——可以通过记录工具选择结果来持续优化
3. **实时性**：工具有效期、权限变更需要即时反映在搜索结果中
