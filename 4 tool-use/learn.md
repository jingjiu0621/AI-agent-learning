# 4 tool-use/ — 工具调用与外部世界交互

## 模块概述

工具调用（Tool Use / Function Calling）是 Agent 与外部世界交互的核心通道。如果说 LLM 是 Agent 的"大脑"，工具就是 Agent 的"手脚"。本模块深入 Function Calling 协议原理、工具定义规范、执行引擎设计、海量工具检索、自定义工具开发，以及多模态/具身智能等高级交互能力。

## 核心思想

1. **工具是 Agent 的能力边界**——Agent 能做多复杂的事，根本上取决于它能调用的工具集
2. **声明式 > 指令式**——用 JSON Schema 声明工具比用自然语言描述更精确、更可靠
3. **工程设计的核心在"执行层"**——工具定义可以几分钟完成，但执行引擎的可靠性、超时、重试、并发决定了生产可行性
4. **海量工具需要检索**——当工具超过 LLM 上下文窗口容量时，语义检索成为必选项
5. **交互形式的演进**：文本 → 结构化输出 → 工具调用 → 多模态 → 具身交互

## 关键工程启示

- 工具描述的质量直接影响 LLM 的选择准确性——投入时间打磨描述比优化模型更有效
- 执行引擎必须有容错设计：超时、重试、优雅降级缺一不可
- 函数调用的"结构"本质上是 LLM 结构化输出的特例——理解 JSON Schema 强制输出是高级用法的基础
- 真实落地时，工具链的安全性（权限/沙箱/审计）往往比功能本身更重要
