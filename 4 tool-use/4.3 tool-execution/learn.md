# 4.3 tool-execution — 工具执行引擎

## 概念定义

工具执行引擎是 Agent 系统中实际运行工具调用的组件。它接收 LLM 输出的函数调用声明，执行对应的函数逻辑，并将结果返回给 LLM 以供下一步推理。执行引擎是 Agent 系统从"思考"到"行动"的桥梁。

## 核心职责

| 职责 | 说明 |
|------|------|
| 函数路由 | 将调用声明映射到具体函数实现 |
| 参数解析 | 解析 LLM 输出的参数，执行类型转换和校验 |
| 安全控制 | 权限检查、沙箱隔离、操作审计 |
| 执行调度 | 同步/异步执行，并行/串行调度 |
| 结果处理 | 结果格式化、错误包装、截断 |
| 生命周期管理 | 超时、重试、回滚、熔断 |

## 设计挑战

1. **异步复杂性**：工具可能耗时数秒到数分钟，需要流式或轮询机制
2. **错误多样性**：网络错误、限流错误、业务逻辑错误、参数校验错误——每种需要不同处理
3. **状态管理**：有状态工具需要跨步骤维护上下文
4. **资源隔离**：某个工具的崩溃不应影响整个 Agent 系统

## 执行模型演进

| 模型 | 说明 | 适用 |
|------|------|------|
| 同步阻塞 | 调一个等一个 | 简单快速工具 |
| 并行执行 | 独立工具同时运行 | 无依赖工具 |
| DAG 调度 | 按依赖图执行 | 复杂工具链 |
| 流式执行 | 边执行边输出 | 长时间任务 |
