# 4.5 custom-tools — 自定义工具开发

## 概念定义

自定义工具开发涉及将真实世界的功能封装为标准化的工具接口，使 LLM Agent 能够调用。这包括文件系统操作、网络请求、代码执行、浏览器自动化等。如果说工具定义是"契约"，自定义工具开发就是"实现"。

## 开发模式

| 层级 | 说明 | 示例 |
|------|------|------|
| Wrapper 层 | 封装已有 API/SDK 为标准工具接口 | 将 Slack SDK 封装为 send_message 工具 |
| 适配层 | 处理认证、分页、重试等横切关注点 | OAuth token 刷新 |
| 执行层 | 实际运行逻辑、资源管理 | HTTP 请求、文件 I/O |
| 安全层 | 沙箱、权限校验、操作审计 | 命令执行前检查白名单 |

## 常见工具类型

| 类型 | 典型工具 | 关键挑战 |
|------|---------|----------|
| 文件 | read/write/search/list | 路径遍历防护、编码处理 |
| 网络 | HTTP GET/POST, 爬虫 | 超时、限流、认证 |
| 代码执行 | Python/SQL 执行 | 沙箱隔离、资源限制 |
| 浏览器 | 截图、点击、填写、导航 | 稳定性、等待策略 |
| 数据库 | 查询、写入、Schema 查看 | SQL 注入、连接管理 |

## 设计原则

1. **幂等性优先**：工具应尽量幂等，便于重试
2. **错误信息友好**：返回结构化错误，让 LLM 能理解并自我纠正
3. **资源边界明确**：每个工具应有明确的超时和资源配额
4. **可观测性**：每个工具有完整的日志和追踪
