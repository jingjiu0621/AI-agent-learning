# 6.4.2 时间衰减（Temporal Decay）

## 简介

时间衰减机制模拟人类记忆的自然遗忘过程，让 Agent 在检索记忆时优先考虑近期内容，同时不彻底丢弃有价值的远期记忆。它是解决记忆随时间"贬值"问题的核心策略。

## 基本原理

时间衰减的核心思想是：每条记忆都有一个随时间递减的时效性权重，检索时将此权重与其他相关性评分相乘，得到最终排序依据。

```
时效分数 F(t) = decay_function(Δt)
其中 Δt = t_current - t_stored
最终得分 = 语义相似度 × F(t) × 重要性权重
```

## 背景与演进

| 阶段 | 方法 | 局限性 |
|------|------|--------|
| 固定窗口 | 只检索最近 N 条记录 | 丢失远期重要记忆 |
| 简单衰减 | 线性或指数衰减 | 无法区分不同重要性的记忆 |
| 自适应衰减 | 根据记忆重要性动态调整衰减率 | 引入了额外的超参数调优需求 |
| 分层次衰减 | 不同记忆层级使用不同衰减策略 | 系统复杂度增加 |

## 核心理念：艾宾浩斯遗忘曲线在 Agent 中的应用

赫尔曼-艾宾浩斯在 1885 年发现人类记忆的遗忘遵循指数衰减规律：刚学完时遗忘最快，之后逐渐放缓。

艾宾浩斯遗忘曲线公式：
```
R(t) = e^(-t / S)
```
其中 R(t) 是保留率，S 是记忆强度。

### 在 Agent 中的映射

| 人类记忆概念 | Agent 对应 |
|-------------|-----------|
| 学习时间 | 记忆创建/更新时间 |
| 记忆强度 | 记忆重要性评分 |
| 复习间隔 | 检索命中次数 |
| 遗忘 | 时效性权重归零 |

Agent 系统借鉴了艾宾浩斯曲线"间隔效应"的思想：越是频繁被检索到的记忆，其衰减速率应当越慢。

## 衰减函数设计

### 1. 指数衰减（Exponential Decay）

```
F(t) = e^(-λ × Δt)
```

- **λ（衰减率）**：控制衰减速度，λ 越大衰减越快
- **半衰期 T½**：F(t) = 0.5 时的时间，T½ = ln(2)/λ
- **特点**：初始衰减极快，长期衰减趋缓，符合遗忘曲线基本形态
- **适用场景**：对话历史记忆中最近的几轮对话权重远高于早期对话

```
示例：
λ = 0.1 时，半衰期 ≈ 6.93 个时间单位
Δt=0 → 1.0    Δt=7 → 0.5    Δt=30 → 0.05
```

### 2. 对数衰减（Logarithmic Decay）

```
F(t) = 1 / (1 + c × log(1 + Δt))
```

- **c**：控制衰减速度的常数
- **特点**：衰减速度远慢于指数衰减，且随时间推移衰减越来越慢
- **适用场景**：长期知识记忆，需要保留远期内容的价值

```
示例：
c=0.5 时
Δt=0 → 1.0    Δt=10 → 0.48    Δt=100 → 0.26    Δt=1000 → 0.15
```

### 3. 线性衰减（Linear Decay）

```
F(t) = max(0, 1 - α × Δt)
```

- **α**：衰减斜率
- **特点**：简单直观，但不够平滑
- **适用场景**：短期缓存、工作记忆中的临时记忆

```
示例：
α=0.01 时
Δt=0 → 1.0    Δt=50 → 0.5    Δt≥100 → 0.0
```

### 四种衰减曲线对比

| 函数 | 初始衰减 | 长期保留 | 控制参数 | 适用场景 |
|------|----------|----------|----------|----------|
| 指数衰减 | 极快 | 趋近于零 | λ（衰减率） | 对话历史、短期交互 |
| 对数衰减 | 较快 | 持续保留 | c（尺度因子） | 知识库、长期经验 |
| 线性衰减 | 匀速 | 完全归零 | α（斜率） | 缓存、临时标记 |
| 幂律衰减 | 快 | 缓慢 | κ, θ | 模拟真实遗忘 |

### 4. 幂律衰减（Power-law Decay）

```
F(t) = (1 + κ × Δt)^(-θ)
```

- **κ, θ**：形状参数
- **特点**：介于指数和对数之间，被神经科学认为是比指数衰减更精确的遗忘模型
- **适用场景**：需要更精细控制衰减曲线的高级系统

## 自适应衰减策略

静态衰减函数无法适应不同记忆的实际价值，自适应衰减通过记忆的访问频率、重要性、检索命中历史动态调整衰减率。

### 检索增强式衰减（Retrieval-Augmented Decay）

每次记忆被成功检索时，重置或削弱其衰减程度：

```
F'(t) = F(t) × (1 - boost)  ← 被检索时
其中 boost 是检索带来的"复习增益"，类似人类复习巩固记忆
```

实现方式：
- **重置法**：被检索后衰减计时器归零，重启衰减
- **减缓法**：被检索后衰减率 λ 降低（如 λ' = λ × 0.9）
- **分级法**：不同检索频率对应不同衰减层级

### 重要性加权衰减

```
λ_effective = λ_base / (1 + γ × importance)
```

- 重要性越高的记忆，衰减越慢
- γ 控制重要性对衰减的影响程度

### 时效性与相关性的联合建模

```
score = sim(q, d) × F(t) × importance × (1 + η × recency_bonus)
```

其中 recency_bonus 是检索命中的历史反馈信号，形成正反馈循环：被检索到的记忆更可能再次被检索到。

## 核心矛盾

**时效性与重要性的矛盾**：短期高频交互产生的记忆往往时效性权重大，但可能并不真正重要；而真正重要的长期知识可能因为时效权重低而被错误地过滤掉。

## 工程优化

- **时间分桶**：将记忆按时间窗口分组（小时/天/周），批量计算衰减权重
- **预计算衰减**：在内存中维护预计算的衰减因子表，避免实时计算
- **冷热分离**：近期记忆（高时效权重）常驻内存，远期记忆存储到磁盘，按需加载
- **延迟衰减**：只在检索时计算衰减，而非实时更新每条记忆的分数

## 能力边界

- 无法仅通过时间衰减区分记忆的语义重要性
- 对时间粒度敏感：秒级衰减不适合长期记忆，天级衰减不适合对话上下文
- 单一衰减函数无法覆盖所有记忆类型的时效特性

## 最大挑战

**遗忘阈值设定**：当衰减权重趋近于零时，是彻底删除记忆还是保留但标记为"归档"？彻底删除可能丢失有价值的远期模式，保留则增加存储成本。需要在存储效率和信息完整性之间做出权衡。

## 场景判断

| 场景 | 推荐衰减策略 |
|------|-------------|
| 多轮对话上下文 | 指数衰减（快衰减，半衰期=5~10轮） |
| 用户偏好记忆 | 对数衰减或检索增强式衰减 |
| 任务执行日志 | 线性衰减（保留当前任务周期内的日志） |
| 长期知识库 | 重要性加权衰减 |
| 行为模式学习 | 幂律衰减 + 检索增强 |
