# 6.4.3 重要性评分（Importance Scoring）

## 简介

重要性评分通过多维评估机制赋予每条记忆一个综合重要性分数，使 Agent 能够在海量记忆中优先关注和保留最有价值的记忆。它与语义检索和时间衰减共同构成记忆检索的三驾马车。

## 基本原理

重要性评分是一个多维度加权求和过程：

```
Score(m) = Σ wᵢ × fᵢ(m)    (其中 fᵢ 是第 i 个评分维度，wᵢ 是对应权重)
```

各维度分数取值范围通常归一化到 [0, 1]，最终综合分数用于驱动记忆的检索排序、存储生命周期管理和遗忘决策。

## 四大核心评分维度

### 1. 频率（Frequency）

反映某条记忆或相似记忆被访问/命中的次数。

```
f_freq(m) = 1 - 1 / (1 + count(m))
```

- **原理**：高频访问的记忆更有价值（类似人类的"常用知识"效应）
- **计算方式**：全局计数器、滑动窗口计数、指数加权移动平均
- **优化方向**：防止频率过高导致的"回音室效应"（重复记忆主导评分）

### 2. 最近性（Recency）

反映记忆距离当前时间有多近。

```
f_rec(m) = decay_function(t_current - t_stored)
```

- **原理**：近期创建或更新的记忆更可能相关
- **与时间衰减的关系**：时间衰减用于检索排序时降低远期记忆权重，而最近性作为重要性维度之一用于评估记忆的"应该被记住"程度
- **区别**：最近性评分是重要性维度，而时间衰减是检索排序策略。重要的远期记忆即使检索时被衰减，其重要性评分仍然高，从而在整体评估中获得优势。

### 3. 相关性（Relevance）

反映记忆与当前查询或上下文的语义匹配程度。

```
f_rel(m, q) = cos_sim(embed(m), embed(q))
```

- **原理**：内容语义越相似的记忆越相关
- **依赖关系**：依赖外部 embedding 模型的编码质量
- **动态性**：每次检索时重新计算，而非静态存储
- **优化方向**：缓存 embedding 向量避免重复编码

### 4. 显著性（Salience）

反映记忆内容的"信息量"或"意外程度"，这是最重要的差异化维度。

```
f_sal(m) = -log(P(content(m)))   ← 信息论视角：低概率事件 = 高信息量
```

- **概率视角**：出现概率越低的事件，显著性越高
- **实现方式**：
  - **统计异常检测**：标记与 Agent 常规行为模式显著偏离的事件
  - **情感强度分析**：情感强烈的事件（正面/负面极端情绪）显著性高
  - **决策分叉点**：导致 Agent 做出重大决策转变的关键记忆
  - **新实体检测**：首次出现的人名、项目名、关键术语

## 加权融合公式设计

### 等权融合（简单平均）

```
Score = (f_freq + f_rec + f_rel + f_sal) / 4
```

- **优点**：实现简单，参数少
- **缺点**：无法区分各维度在不同场景下的重要性差异

### 静态加权融合

```
Score = w₁·f_freq + w₂·f_rec + w₃·f_rel + w₄·f_sal,  Σwᵢ = 1
```

- **权重确定方式**：
  - 领域专家手动设定
  - 基于标注数据的回归学习
  - 网格搜索优化
- **典型权重分配**（对话 Agent）：

| 维度 | 权重 | 说明 |
|------|------|------|
| 频率 | 0.15 | 避免高频噪声过强 |
| 最近性 | 0.25 | 对话场景下近期信息重要 |
| 相关性 | 0.35 | 语义匹配是核心 |
| 显著性 | 0.25 | 异常事件值得关注 |

### 动态加权融合（场景自适应）

根据当前任务的元信息动态调整各维度权重：

```
Score = w₁(t)·f_freq + w₂(t)·f_rec + w₃(t)·f_rel + w₄(t)·f_sal
```

权重向量 W(t) 是当前上下文特征函数：

| 上下文特征 | 权重调整方向 |
|-----------|-------------|
| 对话早期 | 降低最近性权重，提高显著性权重 |
| 知识问答 | 提高相关性权重 |
| 长期规划 | 提高显著性和频率权重 |
| 快速闲聊 | 提高最近性权重 |

### 乘法融合

```
Score = f_freq^α × f_rec^β × f_rel^γ × f_sal^δ
```

- **优点**：任一维度为零则整体归零，天然具备"必要条件"语义
- **缺点**：数值范围敏感，小值维度可能过度主导

## 背景与演进

| 阶段 | 方法 | 局限性 |
|------|------|--------|
| 单维度 | 仅用最近性排序 | 无法识别长期重要记忆 |
| 双维度 | 频率 + 最近性 | 缺乏语义理解 |
| 三维度 | 加入相关性 | 对显著事件不敏感 |
| 四维度 | 频率+最近性+相关性+显著性 | 静态权重泛化能力弱 |
| 自适应 | 动态加权 + 多维度 | 系统复杂度高，调优困难 |

## 核心矛盾

**短期相关性与长期重要性之间的矛盾**：频繁访问和语义高度匹配的记忆在短期内获得高分，但真正定义 Agent 行为特征的"里程碑式"记忆（如一次关键的错误纠正）可能在频率和相关性维度上得分不高，却被显著性维度捕获。

## 工程优化

### 增量更新

- 不需要每次检索都重新计算全部维度
- 频率和最近性可预计算并缓存
- 相关性实时计算（依赖当前查询）
- 显著性可按事件触发更新

### 归一化策略

- **Z-score 归一化**：将各维度分数转化为标准正态分布
- **Min-Max 缩放**：映射到 [0, 1] 区间
- **分位数归一化**：对高度偏态分布更鲁棒

### 重要性衰减

长时间未被检索的记忆，其重要性应随时间缓降，而非永远保持高频：

```
Score'(m, t) = Score(m) × temporal_penalty(Δt)
```

## 能力边界

- 静态权重策略无法适应所有任务类型
- 显著性检测依赖外部 NLP 能力（情感分析、异常检测等）
- 多维度评分增加了系统调优的复杂度
- 高重要性记忆可能因 embedding 的质量问题而被低估

## 最大挑战

**维度间相关性**：四个核心维度并非完全独立。例如，高频记忆往往也是最近被访问的（频率与最近性相关），高显著性的记忆也可能自然获得更高频率。直接线性加权会放大这种相关性，导致评分偏差。需要使用去相关技术（如正交化、PCA 预处理）或非线性融合策略。

## 能力边界与区别对比

| 维度 | 捕获的信息 | 不捕获的信息 | 与时间衰减的区别 |
|------|-----------|-------------|-----------------|
| 频率 | 全局使用模式 | 单次重大事件 | 频率是累计指标，衰减是动态过程 |
| 最近性 | 时间临近性 | 长期趋势 | 最近性作为重要性维度，衰减作为排序策略 |
| 相关性 | 语义匹配度 | 新颖性 | 两者服务于不同阶段（评分 vs 排序） |
| 显著性 | 异常与关键事件 | 日常模式 | 显著性评估事件本身，衰减作用于时间维度 |

## 场景判断

| 场景 | 主导维度 | 权重策略 |
|------|---------|----------|
| 对话式 AI | 最近性 + 相关性 | 较高最近性权重 |
| 知识管理 Agent | 频率 + 显著性 | 较高频率权重 |
| 代码开发助手 | 相关性 + 显著性 | 较高显著性权重 |
| 长时间运行 Agent | 频率 + 显著性 | 动态加权，随时间降低最近性 |
| 个性化推荐 | 频率 + 相关性 | 静态加权，频率权重逐渐增加 |
