# 6.4 记忆检索策略（Memory Retrieval）

## 定义

记忆检索策略决定了 AI Agent 如何从海量存储的记忆中高效、准确地"回忆"出当前上下文最相关的内容。它不仅是简单的查找过程，而是涉及评分、排序、过滤和融合的多阶段决策流程。

## 角色定位

在 Agent 记忆系统中，检索层处于记忆存储与推理执行之间的枢纽位置：

```
外部输入 → 工作记忆 → 记忆检索（当前模块） → 长期记忆 → 推理/决策
                            ↑
                    6.1 短时记忆   6.2 长期记忆   6.3 工作记忆
```

检索策略的质量直接决定 Agent 对历史经验的利用效率，进而影响其决策质量和行为一致性。

## 为什么需要专门的检索策略

- **记忆规模膨胀**：长时间运行的 Agent 可能积累数百万条记忆记录，线性扫描不可行
- **相关性的时变性**：过去重要的记忆现在可能无关，检索必须考虑时间衰减
- **语义复杂性的挑战**：关键词匹配无法捕捉深层语义关系，需要向量化语义检索
- **多维度权衡需求**：精度、延迟、覆盖度三者之间存在根本性矛盾
- **记忆的层次化利用**：不同粒度的记忆（事实、经验、技能）需要不同的检索策略

## 核心矛盾：检索精度 vs 检索延迟 vs 覆盖度

| 维度 | 定义 | 优化方向 | 代价 |
|------|------|----------|------|
| **精度（Precision）** | 检索结果中相关记忆的比例 | 精确匹配、高阈值 | 可能遗漏低相似度但关键的记忆 |
| **延迟（Latency）** | 从发起查询到返回结果的时间 | 索引优化、近似搜索 | 可能降低检索精度 |
| **覆盖度（Recall）** | 所有相关记忆被检索到的比例 | 低阈值、多路召回 | 噪声增加、后处理负担加重 |

这三者构成了检索领域的"不可能三角"——在实际系统中需要根据场景进行有针对性的权衡。

## 四大检索维度概览

| 维度 | 策略 | 核心指标 | 适用场景 |
|------|------|----------|----------|
| **语义检索（6.4.1）** | 向量相似度搜索 | 余弦相似度、MIP | 内容相关的模糊匹配 |
| **时间衰减（6.4.2）** | 时效性加权 | 衰减率、半衰期 | 时效敏感的记忆回忆 |
| **重要性评分（6.4.3）** | 多维加权融合 | 综合得分 | 优先级排序 |
| **多模态检索（6.4.4）** | 跨模态对齐 | 模态间相似度 | 图文/音视频混合记忆 |

## 检索流程总览

```
查询输入
    │
    ▼
┌─────────────────────────────────────┐
│  多路并行检索                        │
│  ├── 语义检索（向量相似度）            │
│  ├── 时间衰减评分                    │
│  ├── 重要性评分                      │
│  └── 多模态匹配（若需要）             │
└──────────┬──────────────────────────┘
           ▼
┌─────────────────────────────────────┐
│  结果融合（6.4.5）                   │
│  ├── RRF 倒数秩融合                 │
│  ├── CC 一致性融合                   │
│  └── 加权聚合                        │
└──────────┬──────────────────────────┘
           ▼
┌─────────────────────────────────────┐
│  阈值过滤（6.4.6）                   │
│  ├── 自适应阈值调整                  │
│  ├── 精度-召回率平衡                 │
│  └── Top-K 截断                     │
└──────────┬──────────────────────────┘
           ▼
       最终记忆输出
```

每个子模块从不同角度解决检索中的特定问题，联合构成一个完整的检索系统。
