# 6.3.1 Scratchpad — 推理草稿纸

## 简单介绍

Scratchpad 是 Agent 在推理过程中记录中间想法、部分结论、候选方案和执行轨迹的临时工作区。它相当于人类解复杂数学题时在草稿纸上演算的过程记录——不追求格式美观，只服务于推理本身。Scratchpad 的存在让 Agent 的推理过程从"黑箱一步到位"变成"可追溯的多步推导"。

## 基本原理

Scratchpad 的核心作用是**分解复杂推理为可追溯的多步链**。在没有 Scratchpad 的情况下，Agent 面对复杂问题需要一次性生成完整答案，这对 LLM 的"一次性推理深度"提出了极高要求。有了 Scratchpad，Agent 可以：

1. **逐步展开推理**：将问题分解为若干推理步，每一步的结果写入 Scratchpad
2. **回溯修正**：当后续推理发现矛盾时，可以回溯到之前的步骤进行修正
3. **暂存中间结论**：复杂推理中的中间变量不需要一次性全部记住，可以写在 Scratchpad 中供后续引用
4. **显式跟踪假设**：记录推理过程中所做的假设和条件分支

Scratchpad 的物理实现通常是一个追加写入的文本缓冲区，每次推理步的输出被附加到缓冲区末尾。LLM 在下一次推理时可以看到完整的 Scratchpad 历史，从而保持推理链的一致性。

## 背景与演进

| 阶段 | 代表工作 | 特点 |
|------|---------|------|
| **无结构 CoT** | Chain-of-Thought (Wei et al., 2022) | 在 prompt 中让 LLM 输出推理过程，但 Agent 架构层不显式管理 |
| **结构化 Scratchpad** | ReAct (Yao et al., 2022) | Scratchpad 开始区分推理轨迹和行动执行结果 |
| **专用缓冲区** | AutoGPT / BabyAGI | 将 Scratchpad 作为独立的内存组件管理 |
| **分层 Scratchpad** | LangChain / CrewAI | 不同粒度、不同可见性的多层 Scratchpad 设计 |

## 核心矛盾

**记录完整性 vs 上下文窗口消耗**：Scratchpad 的一个核心矛盾是——推理过程越详细，Scratchpad 越长，最终会占满上下文窗口。详细记录有利于推理准确性和可调试性，但会加速容量耗尽。

**所有历史都写入 Scratchpad 的"完整记录"策略**与**只保留关键推理链的"摘要记录"策略**之间存在本质张力。前者更可靠但成本高，后者节省空间但可能丢失重要细节。

## 主流优化方向

### 1. 结构化 vs 自由格式

**自由格式 Scratchpad**：以自然语言文本形式记录推理过程，灵活性高，适合开放域推理。

```
用户问题：计算 123 × 456
Scratchpad:
  第一步：分解 456 = 400 + 50 + 6
  第二步：123 × 400 = 49200
  第三步：123 × 50 = 6150
  第四步：123 × 6 = 738
  第五步：求和 49200 + 6150 + 738 = 56088
```

**结构化 Scratchpad**：使用预定义的数据结构（JSON、表格、AST 节点）记录，方便程序化检索和处理。

```json
{
  "step": 3,
  "operation": "multiplication",
  "operands": [123, 50],
  "result": 6150,
  "status": "completed",
  "dependencies": ["step_1"]
}
```

结构化设计便于程序化访问（如按步骤号索引、按状态过滤），但灵活性不如自由格式，对于无法预定义的推理模式可能不适用。

### 2. 可见性控制

**完全可见**：所有 Scratchpad 内容都在 LLM 输入中可见。简单直接，但消耗大量 token。

**部分可见**：只将最近 N 步或最高优先级的 M 条记录送入 LLM 上下文。节省空间但需要策略来决定哪些记录最重要。

**隐式 Scratchpad**：Scratchpad 完全由 Agent 框架管理，LLM 只通过 API 读写特定条目。这种方式最节省 token，但限制了 LLM 的全局推理能力。

### 3. 压缩策略

- **摘要压缩**：用 LLM 或摘要算法将历史推理步骤压缩为更短的摘要
- **结构化压缩**：将文本记录转化为结构化表示（如用数据结构替代自然语言描述）
- **变量替换**：将已经确定的中间结果用简短标识符替代详细推导过程
- **选择性遗忘**：当推理进入新阶段时，主动丢弃已经不再需要的旧 Scratchpad 内容

## 产生的结果

合理使用 Scratchpad 可以带来：
- **推理准确率提升**：多步推理比一步到位的准确率显著更高（在数学推理任务上可达 2-3 倍提升）
- **可解释性增强**：每一步推理都有据可查，便于调试和审计
- **错误定位能力**：当最终答案错误时，可以通过回查 Scratchpad 定位错误的推理步
- **分支探索**：可以在 Scratchpad 中维护多个候选推理路径，选择最优路径继续进行

## 最大挑战

**Scratchpad 污染**：如果之前推理步骤中存在错误或偏见，后续推理会在此基础上继续累积错误，形成"推理链锁定"——一旦早期步骤出错，整个推理链都会偏离正确方向。Scratchpad 的追加特性意味着错误一旦写入就很难"删除"，只能通过"补充修正"来弥补，但 LLM 可能无法有效利用修正信息推翻已建立的推理方向。

## 能力边界

Scratchpad 无法解决以下问题：
1. **需要外部验证的推理**：Scratchpad 只记录推理过程，不验证结果正确性
2. **非语言推理**：空间推理、感知推理等不适合文本记录的推理类型
3. **高度并行的推理**：Scratchpad 本质是线性追加的，不适合记录并行探索的分支
4. **需要实时交互的推理**：Scratchpad 是离线记录，不适合需要实时反馈的动态推理

## 区别对比

| 维度 | 自由格式 Scratchpad | 结构化 Scratchpad |
|------|-------------------|-------------------|
| 灵活性 | 高 | 低 |
| 程序化访问 | 困难 | 便捷 |
| Token 效率 | 低 | 高 |
| 适用场景 | 开放域推理 | 规范化流程 |
| 压缩难度 | 高（语义压缩） | 低（结构压缩） |
| 跨步骤引用 | 模糊（文本描述） | 精确（ID 索引） |

## 核心优势

Scratchpad 最核心的价值在于将 LLM 的推理能力从**单步跳跃**扩展为**多步递推**。这是实现复杂推理 Agent 的基础能力，没有 Scratchpad 的 Agent 在面对需要超过一次推理跳转的问题时几乎必然会失败。

## 工程优化

### 长度控制策略

```python
class Scratchpad:
    MAX_TOKENS = 2048  # 保留的 token 上限

    def add_entry(self, entry: str):
        self.entries.append(entry)
        while self._total_tokens() > self.MAX_TOKENS:
            # 策略1: 丢弃最早的非关键条目
            if not self._is_critical(self.entries[0]):
                self.entries.pop(0)
            # 策略2: 压缩最早的关键条目
            else:
                self.entries[0] = self._compress(self.entries[0])
```

### 检索增强

当 Scratchpad 内容过长时，不是将全部内容送入 LLM，而是用检索机制找到与当前推理最相关的部分：

```
当前问题：需要用到步骤 5-8 的结果
检索策略：
  1. 关键词匹配 -> 定位到步骤 5, 6, 7, 8
  2. 依赖链回溯 -> 额外包含步骤 3, 4（步骤 5 的依赖）
  3. 输出：筛选后的相关条目 -> 送入 LLM 上下文
```

### 快照与恢复

```python
snapshot = scratchpad.snapshot()  # 保存当前状态快照
# ... 尝试某种推理路径 ...
if found_contradiction:
    scratchpad.restore(snapshot)  # 恢复到分歧点，尝试另一条路径
```

## 场景判断

| 场景 | Scratchpad 策略 | 理由 |
|------|----------------|------|
| 数学计算 | 结构化 + 完全可见 | 每一步都依赖前一步结果，需要完整轨迹 |
| 代码生成 | 自由格式 + 摘要压缩 | 需要保留设计思路，但代码骨架会快速增长 |
| 多工具调用 | 结构化 + 部分可见 | 工具调用结果长，只需保留关键输出 |
| 对话历史总结 | 自由格式 + 选择性遗忘 | 只保留语义重要的轮次 |
| 并行搜索探索 | 分叉式 Scratchpad | 每个搜索分支独立的 Scratchpad |
