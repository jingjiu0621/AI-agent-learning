# 6.6 外部记忆（External Memory）

## 定义

外部记忆（External Memory）是指 AI Agent 在自身参数化记忆（模型权重、上下文窗口、内部状态）之外，通过外部系统获取和存储知识的能力。它是 Agent 突破自身记忆边界、接入无限信息空间的桥梁层。

## 角色定位

外部记忆在整个 Agent 记忆体系中扮演"二级存储"角色：

- **对上层**：为工作记忆（Working Memory）和 episodic memory 提供事实依据和长期知识支撑
- **对下层**：对接各种外部数据源，实现知识的持久化、索引化、可追溯化
- **横向**：与内部记忆形成互补——内部记忆负责快速推理和模式匹配，外部记忆负责精确事实和完整上下文

## 为什么需要外部记忆

| 内部记忆的局限 | 外部记忆的弥补 |
|---|---|
| 上下文窗口有限（4K-200K tokens） | 容量仅受存储系统限制，可扩展到 TB 级 |
| 知识截止于训练数据时间点 | 可接入实时数据，知识永不冻结 |
| 模型可能遗忘或幻觉 | 外部存储可精确检索和验证 |
| 单次推理成本随上下文增长 | 按需检索，成本可控 |
| 无法跨会话持久化 | 结构化存储支持跨 session 共享 |

## 核心矛盾：容量 vs. 访问代价

外部记忆面临的根本矛盾是：**信息容量无限，但访问代价高昂**。

- 容量维度：外部存储可以保存任意规模的数据（GB/TB/PB 级），理论上无上限
- 访问代价维度：每次外部读取都需要网络 I/O、序列化/反序列化、索引查找、结果排序，延迟通常在 1ms-500ms 之间，远高于内存访问（纳秒级）

这一矛盾决定了外部记忆架构设计的核心命题：**如何在有限的延迟预算内，从海量存储中精准定位最有价值的信息**。

## 五类外部记忆源概览

### 1. 知识图谱（Knowledge Graph）
基于图结构的语义记忆。将实体（Entity）和关系（Relation）编码为有向图，支持多跳推理和关系查询。

- 代表技术：Neo4j、Amazon Neptune、RDF 三元组存储
- 优势：结构化语义理解、推理能力、可解释性
- 适用：需要精确关系推理的场景（如金融风控、医疗诊断）

### 2. 关系型数据库（Database Memory）
将结构化记忆存储在关系模型中的传统方案。通过 Schema 设计将 Agent 的经验和知识组织为表结构。

- 代表技术：PostgreSQL、MySQL、SQLite
- 优势：强一致性、事务支持、成熟生态
- 适用：需要 ACID 保证的结构化数据存储

### 3. 文档存储（Document Store）
面向非结构化或半结构化文档的记忆存储。支持全文检索和聚合分析。

- 代表技术：MongoDB、Elasticsearch、Couchbase
- 优势：灵活 Schema、强大的搜索能力、水平扩展
- 适用：日志分析、知识库、对话历史存档

### 4. 外部 API 记忆（External API Memory）
通过调用外部服务的 API 接口来获取或存储记忆。这是一种"不存储"的记忆——信息存在于远端系统中。

- 代表技术：RESTful API、GraphQL、gRPC
- 优势：实时性、无需本地存储、接入灵活
- 适用：实时数据查询、第三方系统集成

### 5. 记忆联邦（Memory Federation）
多源记忆的统一查询层。Agent 同时向多个记忆源发起查询，融合结果后返回。

- 代表技术：联邦查询引擎、多模态检索融合
- 优势：综合利用各源优势、查询覆盖面广
- 适用：需要从多个异构数据源获取信息的复杂场景
