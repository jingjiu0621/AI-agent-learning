# 6 memory/ — 记忆系统与知识管理

## 模块定位

记忆系统是 Agent 从"无状态工具"进化为"有经验助手"的关键基础设施。没有记忆的 Agent 每次对话都是"第一次见面"——记不住用户偏好、想不起之前决策、无法从错误中学习。

## 为什么需要记忆系统

### 没有记忆的 Agent 的问题

| 问题 | 表现 | 后果 |
|------|------|------|
| 上下文无持续性 | 每次对话从零开始 | 无法建立长期用户关系 |
| 无经验积累 | 同样的错误反复犯 | 效率低下 |
| 无法个性化 | 每个用户都是陌生人 | 体验差 |
| 知识无法沉淀 | 学到的东西随对话结束丢失 | 无法成长 |

### 记忆解决的核心矛盾

**有限上下文窗口 vs 无限交互历史**：LLM 的上下文窗口有限（即使 200K tokens 也不够），而 Agent 与用户的交互可以持续数月甚至数年。记忆系统解决的就是这个"有限 vs 无限"的根本矛盾。

## 记忆层次架构

Agent 记忆系统借鉴了认知科学的人类记忆模型：

```
                  ┌─────────────────────────────────────┐
                  │            外部记忆（6.6）              │
                  │   知识图谱 / 数据库 / 文档存储 / API   │
                  └────────────────┬────────────────────┘
                                   │ 整合
                  ┌────────────────┴────────────────────┐
                  │          长期记忆（6.2）              │
                  │   向量存储 / 结构化 / KV / 混合      │
                  └────────────────┬────────────────────┘
                                   │ 巩固 ←──────────┐
                  ┌────────────────┴────────────────┐  │
                  │        记忆整合（6.5）            │  │
                  │   摘要 / 抽象 / 去重 / 巩固      │  │
                  └────────────────┬────────────────┘  │
                                   │ 检索              │
                  ┌────────────────┴────────────────┐  │
                  │        记忆检索（6.4）            │  │
                  │   语义 / 时间 / 重要性 / 融合    │──┘
                  └────────────────┬────────────────┘
                                   │ 提取
                  ┌────────────────┴────────────────┐
                  │          工作记忆（6.3）          │
                  │   Scratchpad / 任务栈 / 进度     │
                  └────────────────┬────────────────┘
                                   │ 上下文化
                  ┌────────────────┴────────────────┐
                  │          短期记忆（6.1）          │
                  │    上下文窗口 / Token 管理       │
                  └────────────────┬────────────────┘
                                   │ 注入
                              ┌────┴────┐
                              │  LLM   │
                              └─────────┘
```

## 各子模块关系

| 模块 | 类比人类 | 核心问题 |
|------|---------|---------|
| 6.1 短期记忆 | 正在说的话 | 上下文窗口满了怎么办 |
| 6.2 长期记忆 | 一生经历 | 大量信息怎么存 |
| 6.3 工作记忆 | 手头草稿纸 | 当前任务状态怎么跟踪 |
| 6.4 记忆检索 | 回忆 | 需要时怎么找到对的记忆 |
| 6.5 记忆整合 | 夜间整理 | 碎片怎么变成知识 |
| 6.6 外部记忆 | 笔记本/书籍 | 自己记不住时去哪儿查 |

## 关键设计权衡

1. **存储 vs 检索**：存得越多，找到对的越难
2. **精度 vs 成本**：检索越精确，计算开销越大
3. **实时 vs 批量**：即时整合 vs 空闲批处理
4. **自动 vs 手动**：自主管理记忆 vs 用户控制

## 学习路径建议

```
6.1 短期记忆 → 6.2 长期记忆 → 6.3 工作记忆
                                  ↓
6.6 外部记忆 → 6.4 记忆检索 → 6.5 记忆整合
```

先理解"怎么存"（6.1, 6.2），再理解"怎么跟踪当前状态"（6.3），最后学"怎么取"（6.4）和"怎么优化"（6.5, 6.6）。
