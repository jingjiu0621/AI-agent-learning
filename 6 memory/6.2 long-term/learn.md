# 6.2 长期记忆（Long-term Memory）

## 定义

长期记忆是 AI Agent 的"持久化知识库"，用于跨会话、跨任务地存储、检索和更新信息。与短期记忆（上下文窗口内的临时缓存）不同，长期记忆的生命周期不受单次对话限制，能够跨越数小时、数天甚至数月持续存在，是 Agent 实现持续学习和个性化交互的基础设施。

## 角色与职能

长期记忆在 Agent 架构中承担四个核心职能：

1. **知识持久化**：将对话中产生的有价值信息（用户偏好、领域知识、推理中间结果）从易失的上下文窗口转移到持久存储层。
2. **身份维护**：存储 Agent 的"自我认知"——包括角色设定、行为准则、已完成任务的记录，确保交互风格和目标的一致性。
3. **经验积累**：记录过去成功和失败的决策路径，使 Agent 能够在类似场景下复用经验，避免重复错误。
4. **个性化适配**：维护每个用户的独立画像，实现从"通用 Agent"到"个性化助手"的转变。

## 为什么需要长期记忆

| 问题 | 无长期记忆 | 有长期记忆 |
|------|-----------|-----------|
| 用户重复介绍自己 | 每次会话从零开始 | 自动回忆用户背景 |
| 跨会话任务追踪 | 丢失进度，需要重新确认 | 保持任务状态和上下文 |
| 知识积累 | 每次推理独立，无渐进提升 | 经验可复用，性能持续改善 |
| 多用户区分 | 无法区分不同用户 | 每个用户独立记忆空间 |

## 与人类长期记忆的类比

人类记忆系统按照"编码 — 存储 — 检索"三阶段运作，这一框架直接映射到 Agent 长期记忆的设计：

| 人类记忆阶段 | Agent 对应层 | 技术实现 |
|-------------|-------------|---------|
| 感觉记忆 → 短期记忆 | 上下文窗口 + 工作记忆 | Token 缓存、对话历史 |
| 编码（Consolidation） | 记忆压缩与摘要 | 摘要生成、结构化提取 |
| 存储（Storage） | 长期记忆存储层 | 向量数据库、SQL、KV |
| 检索（Retrieval） | 记忆检索与排序 | 语义搜索、重排序、过滤 |
| 再巩固（Reconsolidation） | 记忆更新与修正 | 覆盖写入、衰减机制 |

## 核心矛盾

长期记忆系统在设计中面临四组根本性矛盾：

### 1. 存储容量 vs 检索精度

- 存储容量越大，检索时需要筛选的候选集越大，精度和速度下降。
- 解决方案包括分层索引（HNSW、IVF）和混合检索策略。

### 2. 持久性 vs 相关性

- 持久保存的信息可能随时间变得过时或不相关。
- 需要在"永不遗忘"和"及时淘汰"之间寻找平衡，引入衰减函数和时间戳加权。

### 3. 写入成本 vs 读取性能

- 高维向量写入涉及计算密集的索引构建，影响吞吐。
- 实时应用需要权衡写入延迟和查询延迟，采用异步写入 + 最终一致性模型。

### 4. 隐私保护 vs 功能完整性

- 持久化用户数据带来隐私合规风险（GDPR、CCPA）。
- 需要内置数据脱敏、加密存储和按需删除的能力。

## 三大主流存储方案概览

| 方案 | 核心思想 | 适合场景 | 代表技术 | 查询方式 |
|------|---------|---------|---------|---------|
| **向量存储** | 将信息编码为高维嵌入向量，通过语义相似度检索 | 非结构化文本、语义搜索、RAG | Chroma, Pinecone, Milvus, Qdrant | 最近邻搜索（ANN） |
| **结构化存储** | 将记忆组织为预定义模式的表结构，通过精确查询检索 | 用户画像、偏好设置、关系明确的元数据 | SQLite, PostgreSQL, DuckDB | SQL 查询、过滤器 |
| **KV 存储** | 以键值对存储简单记忆片段，通过键直接访问 | 缓存、会话状态、简单配置 | Redis, Memcached | O(1) 键查找 |

## 选型决策框架

实际 Agent 系统中，三种方案往往组合使用而非互斥：

```
输入信息
    │
    ├── 高度结构化（用户配置、偏好）→ 结构化存储（SQL）
    ├── 非结构化文本（对话历史、文档）→ 向量存储
    └── 临时高频访问（会话状态）→ KV 存储（Redis）
                        │
                    ┌───┴───┐
                    │ 混合检索 │
                    │ (Hybrid) │
                    └─────────┘
```
