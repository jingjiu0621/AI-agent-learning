# 1.2 history — 从符号主义到 LLM Agent 的演进

## 本节概述

Agent 不是凭空出现的。本节追溯 Agent 从 1950 年代的理论萌芽到 2023+ 年 LLM Agent 爆发的完整演进史，理解每种技术范式的**核心思想、成功原因和失败局限**，从而更深刻地理解当前技术选择背后的历史逻辑。

## 核心思想

1. **历史不是线性进步，而是范式切换**：规则驱动（知识工程）→ 数据驱动（统计学习）→ 意图驱动（预训练+引导）。每次切换都带来能力飞跃，但也意味着旧范式的核心假设被打破
2. **LLM 的涌现能力是意外收获**——不是"设计出来的"而是"规模扩展的自然结果"，这意味着我们仍不完全理解它
3. **老思想在新瓶子里重生**：BDI、SOAR、专家系统等"过时"概念在 LLM 时代以新形式回归

## 关键工程启示

- 理解历史帮助做技术选型：哪些问题是某个范式的固有限制，哪些是可改进的工程问题
- 专家系统的"知识获取瓶颈"在 LLM 时代依然以"数据工程成本"的形式存在
- 范式迁移的速度在加快，保持技术敏感度比精通单一框架更重要
