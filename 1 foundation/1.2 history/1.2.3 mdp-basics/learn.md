# 马尔可夫决策过程与规划算法入门

## 简单介绍

马尔可夫决策过程（Markov Decision Process, MDP）是 AI 中**序列决策问题**的数学框架。它提供了一个形式化的模型来描述 Agent 如何在不确定环境中做出一系列决策——每次决策影响当前收益和未来状态。几乎所有 Agent 问题都可以被建模为 MDP，从下棋到机器人导航到对话系统。

MDP 的核心是"当前状态包含了决定未来所需的所有信息"——马尔可夫性质。

## 基本原理

### MDP 五元组

一个 MDP 由 (S, A, P, R, γ) 五个元素定义：

| 符号 | 名称 | 说明 |
|------|------|------|
| S | 状态空间 | Agent 可能处于的所有状态 |
| A | 行动空间 | Agent 可能采取的所有行动 |
| P(s'\|s,a) | 状态转移概率 | 在状态 s 采取行动 a 后转移到 s' 的概率 |
| R(s,a,s') | 奖赏函数 | 在状态 s 采取行动 a 后转移到 s' 获得的即时奖赏 |
| γ | 折扣因子 [0,1] | 平衡当前奖赏和未来奖赏（越接近 0，越短视） |

### Agent 的目标

最大化**累积折扣奖赏**（Return）：

```
G_t = R_{t+1} + γ·R_{t+2} + γ²·R_{t+3} + ... = Σ_{k=0}^{∞} γ^k·R_{t+k+1}
```

### 规划的核心问题

1. **策略评估**：给定策略 π，计算每个状态的值函数 V^π(s)——按照 π 行动能获得的期望累积奖赏
2. **策略改进**：在当前值函数基础上，找到更好的策略
3. **最优策略**：找到在所有状态上都能获得最高期望累积奖赏的策略 π*

### 关键算法

| 算法 | 类型 | 说明 |
|------|------|------|
| 值迭代 | 动态规划 | 反复更新 V(s) 直到收敛 → 导出最优策略 |
| 策略迭代 | 动态规划 | 评估→改进→再评估→再改进... 循环直至最优 |
| MCTS | 搜索 | 模拟-评估-反向传播，AlphaGo 的核心 |
| 值函数近似 | 函数逼近 | 用神经网络（DQN）等近似 V(s) |

## 背景

- 1950s：Bellman 提出动态规划和 MDP 的数学基础——"Bellman 方程"
- 1960s：Howard 提出策略迭代方法
- 1980s：Sutton 将 MDP 与心理学学习理论结合 → TD 学习
- 1990s：部分可观测 MDP（POMDP）扩展了 MDP 到感知不完全的场景
- 2010s：深度 MDP（DQN, AlphaGo）将 MDP 与深度学习结合

MDP 假设"状态完全可观测"——Agent 知道自己所处的确切状态。当感知不完全时，需要 POMDP（部分可观测 MDP）。

## MDP 在 Agent 工程中的角色

### 显式 MDP（传统方法）
严格定义 S/A/P/R，用值迭代或策略迭代求解：
- ✅ 可证明最优，数学严谨
- ❌ 状态空间爆炸（现实世界状态数量天文数字）
- ❌ 转移概率 P 需要精确建模，几乎不可能

### 隐式 MDP（LLM Agent）
LLM Agent 在没有显式 MDP 建模的情况下进行序列决策：
- LLM 的上下文窗口隐含"状态"
- LLM 通过推理（CoT/ReAct）隐式"规划"
- Prompt 中的指令隐含"奖赏信号"

**关键洞察**：LLM Agent 的强大之处在于它通过预训练内化了"世界如何运作"的知识，不需要显式建模转移概率 P 和奖赏函数 R。

## MDP 概念对 Agent 工程的启发

| MDP 概念 | LLM Agent 中的对应 | 工程含义 |
|---------|-------------------|---------|
| 状态 S | 对话历史 + 环境状态 | 设计良好的状态表示至关重要 |
| 行动 A | 生成的回复/工具调用 | 行动空间设计决定能力上限 |
| 转移 P | 环境的不确定性 | Agent 需要有处理不确定性的机制 |
| 奖赏 R | 任务完成度/用户满意度 | 隐式评估标准，用户体验即奖赏 |
| 折扣 γ | 短期 vs 长期权衡 | 短视 Agent 完成任务快但质量可能差 |
| 策略 π | Agent 的行为模式 | Prompt 定义策略 |
| 值函数 V | Agent 的状态评估 | 反思机制中的自我评估 |

## 工程启发

1. **状态表示设计**：虽然不需要显式 S，但"什么信息进入上下文"实际上等价于 MDP 的状态定义
2. **行动空间设计**：工具定义 = 行动空间；工具的种类和参数直接影响规划质量
3. **长期规划**：γ ≈ 1（长期主义）的任务需要 Plan-and-Execute 模式
4. **不确定性量化**：POMDP 思想——Agent 应知道自己"不知道什么"
5. **多步推理 ≈ 规划**：CoT 本质上是隐式的 MDP 规划——通过"展开"推理步骤找到通往目标的路径

## 适用场景

MDP 形式化分析最适合：
- 理解 Agent 决策问题的结构（状态？行动？不确定性？）
- 评估任务复杂性（状态空间大小？）
- 指导算法选择（确定性环境→规划，不确定性→MCTS/RL）

不适合：
- 直接作为 LLM Agent 的实现框架
- 需要显式建模转移概率的高维场景
