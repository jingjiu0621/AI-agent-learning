# 范式迁移：规则驱动 → 数据驱动 → 意图驱动

## 简单介绍

AI Agent 的发展经历了三次根本性的范式转变：从**规则驱动**（程序员编写明确规则）到**数据驱动**（从数据中学习模式）再到**意图驱动**（直接理解人类意图）。每一次迁移都极大地拓展了 Agent 的能力边界，同时也带来了新的工程挑战。

## 三大范式

### 第一代：规则驱动（Rule-Driven）

**核心思想**：知识可以被显式编码为逻辑规则，Agent 通过执行这些规则来展现智能。

```
程序员：写规则（IF A THEN B），构建知识库
Agent： 执行规则链，推导结论
```

**典型技术**：专家系统、逻辑编程（Prolog）、决策树、状态机

| 维度 | 特征 |
|------|------|
| 知识来源 | 人工编码（知识工程） |
| 推理方式 | 符号推导（前向/反向链） |
| 不确定性 | 通过置信因子处理 |
| 开发方式 | 写规则 |
| 修改方式 | 修改规则 |

**局限**：
- 知识获取瓶颈（KAB）
- 脆弱性（规则覆盖范围外完全失效）
- 维护成本随规则量指数增长

### 第二代：数据驱动（Data-Driven）

**核心思想**：智能来自从大量数据中自动学习模式，而非手工编写规则。

```
数据：准备标注数据集
训练：学习从输入到输出的映射（模式识别）
Agent：对新输入进行预测/分类
```

**典型技术**：机器学习、深度学习、强化学习、推荐系统

| 维度 | 特征 |
|------|------|
| 知识来源 | 标注数据（监督学习）或经验（强化学习） |
| 推理方式 | 统计模式匹配 |
| 不确定性 | 概率输出 |
| 开发方式 | 收集数据 + 训练模型 |
| 修改方式 | 重新训练或微调 |

**局限**：
- 数据依赖：需要大量高质量的标注数据
- 泛化问题：训练分布外的输入表现不可控
- 不可解释性：模型决策过程不透明
- 领域迁移困难：新领域需要新数据

### 第三代：意图驱动（Intent-Driven）

**核心思想**：Agent 直接理解人类的自然语言意图，自主决定如何行动。

```
用户：表达意图（"帮我分析这份报告"）
Agent：理解意图 + 自主规划 + 调用工具
```

**典型技术**：LLM Agent、ReAct、Function Calling

| 维度 | 特征 |
|------|------|
| 知识来源 | 预训练语料 + 上下文注入 |
| 推理方式 | 语言模型推理（CoT/ReAct） |
| 不确定性 | 通过多次采样/Analysis管理 |
| 开发方式 | 写 Prompt + 配置工具 |
| 修改方式 | 修改 Prompt + 更新工具 |

## 范式对比

| 维度 | 规则驱动 | 数据驱动 | 意图驱动 |
|------|---------|---------|---------|
| **智能来源** | 手工编码 | 数据学习 | 预训练模型 |
| **灵活性** | 极低 | 中等 | 极高 |
| **维护成本** | 高（规则冲突） | 中等（数据质量） | 中低（Prompt 迭代） |
| **可解释性** | 好 | 差 | 中等 |
| **确定行为** | 是 | 否 | 部分 |
| **部署成本** | 低 | 高（训练） | 中（API） |
| **开发效率** | 慢 | 慢 | 快 |
| **应用广度** | 窄 | 中 | 广 |
| **颠覆性** | 1970s-80s 主导 | 2010s 主导 | 2023+ 主导 |

## 历史演进路径

```
规则驱动                    数据驱动                    意图驱动
─────────                  ─────────                  ─────────
    │                          │                          │
    ▼                          ▼                          ▼
  专家系统                  神经网络                  LLM Agent
  知识工程                  特征工程                  Prompt 工程
  确定性规则                统计模式                  自然语言
  人工构建                  自动学习                  预训练+引导
                                                           
    │                          │                          │
    └──────────────────────────┴──────────────────────────┘
                            时序
            1970 ──── 2010 ──── 2015 ──── 2022 ────→
```

## 核心矛盾

| 矛盾 | 说明 |
|------|------|
| **自由度 vs 可控性** | 意图驱动自由度最高，但行为可控性最差 |
| **泛化 vs 精准** | 规则驱动最精准但泛化最差，数据驱动泛化好但不够精准 |
| **开发快 vs 调试难** | 意图驱动开发最快，但调试和排查问题最困难 |
| **低门槛 vs 高滥用风险** | 意图驱动让非技术人员也能构建"AI 应用"，但安全隐患更大 |

## 范式转变的工程影响

### 角色变化
| 角色 | 规则驱动 | 数据驱动 | 意图驱动 |
|------|---------|---------|---------|
| 开发者角色 | 知识工程师 | 数据科学家/ML工程师 | Prompt 工程师/Agent架构师 |
| 核心技能 | 逻辑/规则编写 | 特征/模型调优 | Prompt 设计/工具编排 |
| 调试工具 | 规则追踪器 | 可视化/实验跟踪 | 链路追踪/日志回放 |

### 工具链变化
| 阶段 | 工具 |
|------|------|
| 规则驱动 | 规则引擎（Drools）、Prolog IDE |
| 数据驱动 | Jupyter、TensorFlow、MLflow |
| 意图驱动 | LangChain、LangSmith、OpenAI SDK |

### 评估方式
| 阶段 | 评估方式 |
|------|---------|
| 规则驱动 | 规则覆盖率、正确率 |
| 数据驱动 | 准确率、F1、AUC |
| 意图驱动 | 任务完成率、用户满意度、多维度 LLM 评估 |

## 工程启示

1. **三范式共存**：新范式不意味着完全替代旧范式——最佳实践中三者在不同层次共存
   - 规则驱动：安全约束、权限控制、输入验证
   - 数据驱动：推荐、分类、预测等需要精确统计的任务
   - 意图驱动：对话、规划、多步推理等需要灵活性的任务

2. **逐步迁移而非全面替换**：从规则/数据驱动逐步扩展意图驱动能力，而不是一次性重构

3. **保留确定性的锚点**：即使在意图驱动的系统中，关键安全检查仍应该是规则化的

4. **工具链升级**：从 MLflow 类实验跟踪升级到 Agent 行为追踪（LangSmith 等）
