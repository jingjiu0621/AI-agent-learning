# 早期基于规则的专家系统（MYCIN, DENDRAL）

## 简单介绍

基于规则的专家系统是 AI 最早成功落地的 Agent 形式。它使用"IF-THEN"规则编码人类专家的领域知识，通过推理引擎执行逻辑推导来解决特定领域问题。MYCIN（医学诊断）、DENDRAL（化学分析）等系统是这一范式的开创者和代表性成果。

## 基本原理

```
┌─────────────────────────────────────────┐
│              专家系统                        │
│                                              │
│  ┌──────────┐    ┌──────────────┐           │
│  │  知识库    │    │  推理引擎      │           │
│  │ (规则集)   │◄───│  前向/反向链    │           │
│  └──────────┘    └──────┬───────┘           │
│       ▲                  │                   │
│       │                  ▼                   │
│  ┌────┴─────┐    ┌──────────────┐           │
│  │ 事实数据库 │    │  用户界面      │           │
│  └──────────┘    └──────────────┘           │
└─────────────────────────────────────────┘
```

- **知识库**：以 IF-THEN 规则形式存储的领域专业知识，形如 `IF 症状A AND 症状B THEN 诊断 = 疾病C (置信度 0.8)`
- **推理引擎**：控制规则的执行顺序，分为前向链（数据驱动：已知事实 → 推导结论）和反向链（目标驱动：假设结论 → 寻找支持事实）
- **事实数据库**：当前案例的具体信息，即推理的起点

## 背景

1950-60 年代 AI 研究者乐观地认为，只要将足够的"常识"编码为逻辑规则，就能实现通用人工智能。但现实是：

1. 常识规模远超预期（McCarthy 的"常识问题"）
2. Feigenbaum 等人提出了"知识原则"：系统的能力来源于它拥有的知识，而非推理机制

这催生了**知识工程**（Knowledge Engineering）——专注于获取和编码人类专家的领域知识。1970 年代，专家系统进入黄金期：

- **DENDRAL**（1965）：化学质谱分析，推断分子结构
- **MYCIN**（1976）：血液感染诊断，约 600 条规则，诊断准确率超过部分医生
- **XCON/R1**（1980）：DEC 计算机配置，每年为 DEC 节省数千万美元

## 之前做法的结果

### 成就
- 在特定领域达到甚至超越人类专家水平
- 首次证明了 AI 系统可以产生商业价值
- 建立了知识工程的方法论和工具链

### 局限
- **知识获取瓶颈（Knowledge Acquisition Bottleneck）**：从专家那里提取并编码规则极其耗时，专家自己往往说不清自己是怎么决策的
- **脆弱性（Brittleness）**：遇到规则覆盖范围外的输入时，系统完全失败
- **维护困难**：规则间存在隐含依赖，修改一条规则可能产生不可预见的副作用
- **缺乏常识**：没有"背景知识"，不理解"发烧通常意味着身体有问题"这类常识
- **无法学习**：经验不会自动转化为知识库的改进

## 核心矛盾

**知识获取瓶颈与系统性能的矛盾**：
- 要覆盖更多场景 → 需要更多规则 → 知识获取成本指数增长
- 规则数量增长 → 规则间冲突可能性增加 → 维护难度剧增

这个矛盾最终限制了专家系统的可扩展性，使其难以走出受限的领域。

## 专家系统 vs LLM Agent

| 维度 | 专家系统 | LLM Agent |
|------|---------|-----------|
| 知识来源 | 手工编码 | 预训练 + 上下文学习 |
| 推理方式 | 规则链（精确） | 概率推理（统计性） |
| 知识更新 | 手工添加/修改规则 | 新 Prompt / RAG / 微调 |
| 领域覆盖 | 窄领域、高精度 | 宽领域、中等精度 |
| 可解释性 | 可追踪推理链 | 半可解释 |
| 维护成本 | 高（规则冲突管理） | 中（Prompt 管理） |
| 脆弱性 | 边界外完全失败 | 边界外仍可尝试（质量下降） |

## 专家系统的遗产

虽然纯专家系统已被 LLM Agent 取代，但其设计理念深刻影响了现代 Agent 架构：

1. **知识库 → 向量数据库/RAG**：将规则存储转化为可检索的知识索引
2. **推理引擎 → LLM 推理链**：前向链/反向链的思想影响了 CoT 的推理模式
3. **规则确定性 → 约束强制执行**：专家系统的确定性规则演变为 Agent 的约束条件和输出护栏
4. **知识工程 → Prompt Engineering**：从"如何编码知识"变为"如何描述知识"

## 工程启示

现代 Agent 设计仍可以从专家系统中汲取经验：
- **规则适用场景**：不需要 LLM 判断的确定逻辑（如输入验证、权限控制）仍用规则实现
- **规则与 LLM 的混合**：精确约束用规则，模糊判断用 LLM
- **知识分层**：专家的"深层知识"（原理）和"表面知识"（经验规则）在 Agent 中应分别处理
