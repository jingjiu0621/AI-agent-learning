# 强化学习 Agent（DQN, PPO, AlphaGo）

## 简单介绍

强化学习（Reinforcement Learning, RL）Agent 通过学习"在什么状态下采取什么行动可以最大化累积奖赏"来掌握决策能力。与专家系统从人类获取知识不同，RL Agent 通过**和环境交互试错**来学习——这种学习方式更接近人类和动物的自然学习过程。

## 基本原理

### 标准 RL 交互框架

```
Agent                    环境
┌─────┐                  ┌─────┐
│ 状态 │◄──观测(O,t)──────│     │
│ Sk  │                  │     │
│     │────行动(At)─────►│     │
│ 策略 │◄──奖赏(Rt+1)────│     │
│ π    │                  │     │
└─────┘                  └─────┘
```

**核心要素**：
- **策略 π(a|s)**：Agent 在状态 s 时选择行动 a 的概率分布——这是 Agent 要学习的目标
- **值函数 V(s)/Q(s,a)**：评估状态或"状态-行动对"的好坏（期望累积奖赏）
- **奖赏信号 R**：环境对 Agent 行动的即时反馈，驱动学习的方向
- **探索 vs 利用**：Agent 需要平衡"尝试新行动"（探索）和"使用已知最优行动"（利用）

### 关键算法

**DQN（Deep Q-Network, 2013）**：
- 用深度神经网络近似 Q 函数 Q(s,a)
- 通过经验回放（Experience Replay）打破数据相关性
- 解决了高维状态空间（如游戏画面）的问题

**PPO（Proximal Policy Optimization, 2017）**：
- 在策略梯度方法基础上引入"裁剪目标函数"，限制每次更新步长
- 训练更稳定，是目前最广泛使用的 RL 算法之一

**AlphaGo（2016）**：
- 结合深度神经网络 + 蒙特卡洛树搜索（MCTS）
- 策略网络（选择行动）+ 值网络（评估局面）+ MCTS（规划）
- 击败围棋世界冠军李世石

## 背景

RL 的发展脉络：

1. ** Trial-and-Error 学习**：Thorndike (1911) 效果律
2. **最优控制**：Bellman (1957) 动态规划，值迭代/策略迭代
3. **时间差分学习**：Sutton (1988) TD 学习
4. **Q 学习**：Watkins (1989) 无模型 RL
5. **深度 RL 革命**：Mnih (2013) DQN 在 Atari 游戏上超越人类
6. **策略梯度**：Schulman (2015/2017) TRPO / PPO
7. **RL+搜索**：Silver (2016) AlphaGo
8. **RL+LLM**：RLHF（RL from Human Feedback）——用 RL 对齐语言模型

## 之前做法的结果

### 成就
- 游戏领域：Atari 游戏、围棋（AlphaGo）、Dota（OpenAI Five）、星际争霸（AlphaStar）
- 机器人控制：机械臂操作、双足行走
- 工业优化：数据中心冷却（Google）、芯片布局（AlphaChip）
- NLP 对齐：RLHF 使 LLM 与人类偏好对齐

### 局限
- **样本效率低**：DQN 在 Atari 上需要数千万帧训练，AlphaGo 需要数百万局自对弈
- **奖赏设计困难**：复杂任务中设计合适的奖赏函数极具挑战，稀疏奖赏让学习几乎不可能
- **泛化能力弱**：在训练环境中学到的策略很难迁移到新环境
- **安全问题**：Agent 可能通过"投机取巧"最大化奖赏，而非学习预期行为
- **计算成本**：大规模 RL 训练需要大量 GPU/TPU 资源

## 核心矛盾

**探索 vs 利用**：
- 探索不够 → Agent 陷入次优解
- 探索过多 → 效率低下，浪费算力

**奖赏工程 vs 真实目标**：
- 奖赏函数是真实目标的不完美代理
- 设计缺陷导致 Agent 出现"奖励黑客"（Reward Hacking）行为

## RL Agent vs LLM Agent

| 维度 | RL Agent | LLM Agent |
|------|---------|-----------|
| 知识来源 | 试错经验 | 预训练语料 |
| 训练方式 | 与环境交互学习 | 语言建模预训练 |
| 推理方式 | 学到的策略直接映射 | 上下文推理+规划 |
| 适应方式 | 需要重新训练或微调 | 通过 Prompt/RAG 即可适应 |
| 样本效率 | 极低 | 高（零样本或少样本学习） |
| 长期规划 | 通过折扣因子隐式建模 | 通过 CoT/ReAct 显式推理 |
| 组合能力 | 弱（策略固化） | 强（工具组合） |
| 工程复杂度 | 高（训练基础设施） | 中（API 调用+Prompt 管理） |

## RL 对 Agent 工程的启示

1. **RLHF 是 LLM Agent 对齐的关键工具**：通过偏好学习使 Agent 的行为符合人类期望
2. **探索策略启发 Agent 的反思机制**：Reflexion 中的自我评估类似于 RL 中的值函数
3. **MCTS 思想在推理中的应用**：Tree of Thoughts 中搜索推理路径类似于 MCTS
4. **奖赏建模用于 Agent 评估**：设计 Agent 的评分标准如同设计奖赏函数
5. **策略梯度优化 Auto-Prompt**：DSPy 等自动 Prompt 优化受策略梯度方法的启发

## 工程启示

- **不从头训练**：现代 Agent 不需要从头训练策略，用 LLM 的推理能力替代策略学习
- **RL 作为微调工具**：只在需要对齐特定偏好时使用 RLHF
- **探索作为推理**：将 RL 中的"探索"概念转化为 Agent 的"多路径推理"或"假设检验"
- **奖赏作为评价**：用精心设计的评估指标（而非奖赏信号）来指导 Agent 的行为
