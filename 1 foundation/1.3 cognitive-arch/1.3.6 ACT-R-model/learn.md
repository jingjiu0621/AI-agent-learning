# ACT-R 认知架构与工作记忆模型

## 简单介绍

ACT-R（Adaptive Control of Thought—Rational）是由 John Anderson 开发的认知架构，基于认知心理学的实验数据构建，旨在**模拟人类认知过程**。与 SOAR 的"通用智能"目标不同，ACT-R 更注重认知的**心理学真实性**——它的每个模块都对应人脑的特定认知功能，参数通过实验数据校准。

## 基本原理

### ACT-R 的核心架构

```
                    ┌─────────────┐
                    │   意图模块    │  ← 控制当前目标和子目标
                    │ (Goal Module)│
                    └──────┬──────┘
                           │
┌─────────┐    ┌──────────┴──────────┐    ┌─────────┐
│ 视觉模块  │───→│    工作记忆中心       │←───│ 手动模块  │
│ (Visual)  │    │  (Working Memory)   │    │ (Manual) │
└─────────┘    └──────────┬──────────┘    └─────────┘
┌─────────┐               │               ┌─────────┐
│ 听觉模块  │───→┌────────┴────────┐←───│ 发声模块  │
│ (Aural)  │    │   产生式系统      │    │ (Vocal)  │
└─────────┘    │ (Production      │    └─────────┘
               │   System)         │
┌─────────┐    └────────┬────────┘    ┌─────────┐
│ 声明式记忆│←───────────┘───────────→│ 过程性记忆│
│ (Declarative)│                       │ (Procedural)│
└─────────┘                           └─────────┘
```

### ACT-R 的关键模块

| 模块 | 功能 | LLM Agent 对应 |
|------|------|---------------|
| 视觉/听觉模块 | 外部世界感知 | 多模态输入处理 |
| 声明式记忆 | 事实知识（"知道什么"） | 参数知识 + RAG |
| 过程性记忆 | 操作知识（"知道怎么"） | System Prompt 规则 |
| 工作记忆 | 当前思考内容 | 上下文窗口 |
| 意图模块 | 目标跟踪 | 任务栈 + 进度跟踪 |
| 手动/发声模块 | 行动输出 | 工具调用输出 |

### 核心机制

**产生式系统**：
- IF-THEN 规则匹配工作记忆内容
- 多条规则可匹配时通过"效用"选择
- 执行规则产生行动或改变工作记忆

**声明式记忆的激活机制**：
- 记忆片段有"基级激活"（使用频率）和"关联激活"（上下文相关性）
- 激活度超过阈值 → 被检索到
- 激活度衰减 → 遗忘

**工作记忆容量限制**：
- 人类工作记忆约 7±2 个组块
- 新内容进入 → 旧内容可能被挤出

## 背景

ACT-R 的发展始于 1970 年代 Anderson 的 ACT 理论，经过了 ACT* → ACT-R → ACT-R 5.0 → ACT-R 7.0 的演进。与 SOAR 不同：

- **SOAR 目标**：展示通用智能
- **ACT-R 目标**：精确模拟人类认知过程（包括限制和缺陷）

ACT-R 最独特之处是它的**参数化**——每个模块都有可调参数，通过拟合人类行为实验数据来确定参数值。

## ACT-R 对 Agent 工程最有价值的贡献

### 1. 工作记忆模型

ACT-R 对工作记忆的建模方式对 Agent 设计有直接启发：

| 概念 | ACT-R | Agent 工程对应 |
|------|-------|---------------|
| 容量限制 | 7±2 组块 | 上下文窗口 Token 预算 |
| 激活扩散 | 一种内容激活另一种 | 注意力机制 |
| 时间衰减 | 不用的内容激活度降低 | 滑动窗口 + 摘要 |
| 干扰 | 新内容覆盖旧内容 | Token 溢出截断 |

**工程应用**：
- 上下文窗口不应该装满，预留 Token 给可能的反馈
- 频繁引用的信息保持"高激活"（永远保留在上下文）
- 长时间未用的信息自动迁移到长期记忆

### 2. 声明式记忆的检索

ACT-R 的记忆检索机制类似于现代 RAG 系统：
- **基级激活** → 语义检索的静态相关性分
- **关联激活** → 上下文动态加权
- **激活阈值** → 检索的最低相似度阈值

### 3. 产生式系统的竞争

多条规则同时匹配当前状态时，ACT-R 通过"效用"（Utility）选择：

**工程应用**：
- 多个工具都适用于当前场景时，依据历史成功率和成本选择
- Agent 的"工具选择"可以建模为产生式竞争

## ACT-R vs LLM Agent 的差异

| 维度 | ACT-R | LLM Agent |
|------|-------|-----------|
| 认知目的 | 模拟人类 | 完成任务 |
| 知识编码 | 产生式规则 | 神经网络参数 |
| 学习方式 | 产生式调试+参数调优 | 预训练+微调 |
| 模块性 | 强模块化 | 整体化 |
| 参数基础 | 心理实验数据 | 大规模语料 |
| 实现复杂度 | 高（需领域建模） | 中（API 调用） |

## 对 Agent 设计的启发

1. **承认容量限制并管理它**：而不是试图扩展它——就像 ACT-R 模拟人类工作记忆限制一样
2. **产生式规则的现代对应**：产生式规则 → Prompt 中的约束规则（"如果 X 情况，做 Y"）
3. **记忆激活的工程化**：高频信息保活、低频信息归档
4. **认知负载最小化**：减少 Agent 需要同时"记住"的信息量，降低错误率

## 能力边界

- ACT-R 的完整实现极其复杂，不适合直接用于工程
- 其最大价值在于**设计启发**而非直接实现
- ACT-R 的"人类认知模拟"目标与 Agent 的"高效任务完成"目标有本质差异
