# 行动模块：工具调用、输出生成与环境交互

## 简单介绍

行动（Act）是 Agent 认知循环的输出端——将思考阶段的决策转化为实际操作。在 LLM Agent 中，行动主要包括：**调用外部工具**（API/代码执行/搜索）、**生成回复内容**（回答/报告/消息）和 **改变环境状态**（写文件/操作数据库/控制设备）。

## 基本原理

### 行动分类

```
                    ┌────── 生成回复（文本输出）
                    │
行动 ──── 输出 ─────┼────── 生成结构化数据（JSON/代码）
         类         │
                    └────── 生成多媒体（图像/音频）
          
                    ┌────── 调用 API（Function Calling）
                    │
行动 ──── 工具 ─────┼────── 代码执行（Python/SQL/Shell）
         调用       │
                    ├────── 文件操作（读/写/搜索）
                    │
                    └────── 多 Agent 通信（发送消息）
                    
                    ┌────── 确认/投票
         系统 ──────┤
         操作       └────── 状态变更
```

### 行动的生命周期

```
思考结果 → 行动选择 → 行动执行 → 结果处理
    │          │           │          │
    ▼          ▼           ▼          ▼
"需要查   LLM 选择   实际调用    解析返回值
 天气"    天气工具   天气 API    注入上下文
```

## 背景

传统程序的"行动"非常直接：函数调用 → 执行 → 返回结果。Agent 的行动复杂性在于：

1. **工具数量不确定**：少到几个、多到数百个工具，选择是关键
2. **执行不确定性**：工具可能失败、超时、返回异常结果
3. **多步行动依赖**：后续行动依赖于前面的执行结果
4. **异步与并发**：多个工具可以并行调用

## 核心挑战

### 1. 工具选择的准确性
- **问题**：工具多时 LLM 可能选错工具或生成错误的参数
- **原因**：工具描述不够清晰、工具间功能重叠
- **方案**：工具描述优化（Description Engineering）、工具检索（海量工具场景）

### 2. 执行可靠性
- **问题**：工具调用可能因网络/认证/超时等原因失败
- **方案**：重试机制 + 降级策略 + 替代工具

### 3. 并发与依赖
- **问题**：多个独立工具可以并行调用以提高效率
- **方案**：并行执行（独立工具）+ 顺序执行（依赖工具）

### 4. 安全控制
- **问题**：工具调用有安全风险（数据泄露/误操作）
- **方案**：权限控制 + 高危操作确认 + 审计日志

## 工程实现

### 行动执行引擎架构

```
┌─────────────────────────────────────────┐
│              行动执行引擎                    │
├─────────────────────────────────────────┤
│  行动规划 → 调度器 → 执行器 → 结果聚合    │
│     │          │       │          │     │
│     ▼          ▼       ▼          ▼     │
│  依赖图分析   并行/   安全沙箱   结果验证   │
│             顺序决策                      │
└─────────────────────────────────────────┘
```

### 常见行动模式
1. **单一工具调用**：一次调用一个工具，获取结果
2. **并行工具调用**：多个独立工具同时调用（Function Calling 中的 parallel FC）
3. **链式工具调用**：B 的结果依赖于 A 的输出，顺序执行
4. **条件工具调用**：根据中间结果决定是否继续调用工具

### 关键实现要点
1. **工具执行的超时控制**：每个工具设定超时时间，避免卡死
2. **结果大小限制**：大型结果自动截断/摘要
3. **错误分类处理**：区分可重试错误（网络超时）和不可重试错误（参数无效）
4. **审计日志**：记录每次工具调用的入参、出参、耗时

## 优化方向

1. **行动预测**：根据当前上下文预测下一步可能需要执行的行动，提前准备
2. **行动组合**：将常见的工具调用序列固化为"组合行动"，提高效率
3. **流式行动反馈**：长时间运行的工具提供进度反馈，避免 Agent 等待超时
4. **自适应并行度**：根据工具依赖和当前运行环境动态调整并行数量
5. **行动回滚**：支持撤销/回滚已完成的操作（符合 ACID 原则）
