# 反馈循环：观察结果驱动下一轮思考

## 简单介绍

反馈循环（Feedback Loop）是 Agent 认知循环的关键——Agent 行动后，环境状态发生了变化，产生了新的观察结果，这些结果被感知模块捕获，驱动下一轮的思考和行动。这个循环正是 Agent 能够自主执行多步任务的根本机制。

## 基本原理

### 反馈循环的完整流程

```
         ┌────────────────────────────────────┐
         │                                    │
         ▼                                    │
    ┌────────┐    ┌────────┐    ┌────────┐    │
    │ 感知    │───→│ 思考    │───→│ 行动    │───┘
    │(Observe)│    │(Think) │    │(Act)   │
    └────────┘    └────────┘    └────────┘
         ▲                                    │
         │                                    │
         └──────────── 环境变化 ──────────────┘
```

### 反馈的类型

| 反馈类型 | 来源 | 示例 |
|---------|------|------|
| 工具返回值 | 工具执行结果 | API 返回的数据 |
| 执行状态 | 执行成功/失败 | 错误码、异常信息 |
| 环境状态 | 外界系统变化 | 新消息到达、文件变更 |
| 用户反馈 | 用户对 Agent 的响应 | 纠正、确认、新指令 |
| 内部状态 | Agent 自身评估 | 反思结果、置信度 |

## 背景

反馈循环是"Agent vs 传统程序"的核心区别之一：

- **传统程序**：执行完就结束了，输出结果固定
- **Agent**：执行行动 → 观察结果 → 决定下一步（可能改变原计划）

反馈循环使 Agent 具备了**适应性**——Agent 不是机械地执行预定计划，而是根据实际结果动态调整。

## 核心挑战

### 1. 结果的准确理解
- **问题**：工具返回结果可能模棱两可或包含噪声
- **方案**：结果解析 + 格式验证 + LLM 辅助理解

### 2. 反馈与目标的关联
- **问题**：Agent 需要判断"这个结果可靠目标更近了还是偏离了"
- **方案**：状态评估机制（检查当前进展与目标的差距）

### 3. 循环终止
- **问题**：Agent 可能陷入无限循环（不断调用工具但无法推进任务）
- **方案**：最大步数限制 + 进度检测 + 循环检测

### 4. 错误的积累
- **问题**：每轮循环中的小错误会积累放大
- **方案**：定期检查 + 错误纠正 + 关键时刻回溯

## 工程实现

### 反馈处理的 Pipeline

```
工具返回 → 格式验证 → 内容解析 → 重要性评估 → 上下文注入
                     │                       │
                     ▼                       ▼
                 错误处理                 Token 预算
                 (重试/降级)            (摘要/截断)
```

### 循环控制策略

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| 固定步数 | 最多执行 N 步 | 简单任务 |
| 动态终止 | 检测到目标完成即停止 | 复杂任务 |
| 超时终止 | 超过时间限制 | 实时场景 |
| Token 预算 | 达到 Token 上限 | 成本敏感 |
| 循环检测 | 检测到重复行为模式 | 所有场景 |

### 关键实现要点
1. **结果缓存**：短期缓存工具执行结果，避免重复调用
2. **进度评估**：每轮循环评估"已完成%"和"剩余步骤估算"
3. **循环熔断**：检测到重复模式时主动中断并提示
4. **中间结果持久化**：重要中间结果保存到工作记忆中

## 优化方向

1. **异步反馈处理**：不阻塞等待反馈，边执行边处理已返回的结果
2. **反馈优先级排序**：多路反馈同时到达时，按重要性排序处理
3. **预期与实际对比**：Agent 在行动前预测返回结果，与实际结果对比检测异常
4. **选择性反馈**：Agent 只关注与任务相关的环境变化，忽略无关信息
