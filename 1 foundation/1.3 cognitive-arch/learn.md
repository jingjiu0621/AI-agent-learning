# 1.3 cognitive-arch — 认知架构：感知 → 思考 → 行动 循环

## 本节概述

理解了"Agent 是什么"和"Agent 从哪来"之后，本节深入 Agent 的内部工作方式——**认知架构**。核心循环（感知→思考→行动→反馈）是所有 Agent 系统的基本模式，而 SOAR 和 ACT-R 这两个经典认知理论为理解这一循环提供了深厚的理论支撑。

## 核心思想

1. **认知循环是 Agent 的基本节拍**——所有 Agent 都遵循 "Observe → Orient → Decide → Act" 的循环模式
2. **模块分离是工程最佳实践**——将感知、推理、行动模块分离，便于各自独立优化和测试
3. **SOAR 的"当机"概念最有价值**——Agent 不确定时应创建子目标来获取信息，而非胡乱猜测
4. **ACT-R 的工作记忆模型是"减法的智慧"**——承认容量限制并管理它，比试图扩展容量更实用

## 关键工程启示

- 反馈循环驱动 Agent 的适应能力——没有反馈的 Agent 只是单次 API 调用
- 工作记忆管理是 Agent 工程的核心挑战：什么留在上下文，什么移到外部存储？
- 认知架构理论的价值在于"设计启发"而非"直接实现"
