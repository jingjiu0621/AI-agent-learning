# CrewAI：角色分工与层级协作

## 简单介绍

CrewAI 是一个专注于**多 Agent 协作**的框架，设计哲学是"角色分工"——开发者定义不同的 Agent 角色（如"研究员"、"分析师"、"写手"），分配任务，CrewAI 负责 Agent 之间的通信和任务协调。

## 核心设计

```
Agent: 角色 + 目标 + 技能 → 配置每个 Agent 的 "人格"
Task: 描述 + 期待输出 + Agent 分配
Crew: Agent 团队 + 执行流程
Process: Sequential(顺序) | Hierarchical(层级)
```

## 核心优势

| 优势 | 说明 |
|------|------|
| 角色设计直观 | Agent 角色定义清晰，适合模拟团队协作 |
| 内建流程 | Sequential / Hierarchical 两种流程即开即用 |
| 任务委派 | 自动将任务分配给合适的 Agent |
| 结果聚合 | 自动汇总多 Agent 的输出 |

## 主要不足

| 不足 | 说明 |
|------|------|
| 定制化有限 | 流程图式固定，复杂控制流受限 |
| 单 Agent 弱 | 对单 Agent 场景支持不如 LangChain |
| 开销 | 多 Agent 调用有额外 Token 和延迟开销 |
| 调试难 | Agent 间通信链追踪不直观 |

## 适用场景

- **内容生产 pipeline**：研究 → 分析 → 写作的多步内容生成
- **多角色分析**：从不同视角分析同一个问题
- **模拟与游戏**：角色扮演场景

## 不适用场景

- 单 Agent 简单任务
- 需要细粒度控制流的复杂工作流
