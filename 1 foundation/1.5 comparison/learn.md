# 1.5 comparison — 主流框架对比与选型

## 本节概述

有了 Agent 的基础认知后，本节对比当前主流 Agent 框架（LangChain/LangGraph、CrewAI、AutoGen）和手写方案的优劣势，提供一个**框架选择的决策框架**，并对比各方案的性能和开发效率，帮助在实际项目中做出合理选择。

## 核心思想

1. **框架解决的是"规模问题"**——工具少（<5）、步骤少（<10）时手写更优
2. **没有免费的抽象**——框架每层抽象都带来性能/Token 开销和调试复杂度
3. **混合架构是最优解**——核心逻辑手写 + 非核心用框架工具
4. **选择框架前先理解原理**——手写一个最小 Agent 是正确学习路径的起点

## 关键工程启示

- 对于学习路径：先手写（理解原理）→ 再框架（解决规模）
- 框架选择的核心问题：它对 **Prompt 的控制力** 和 **状态管理的灵活度**
- 生产环境中，可观测性（LangSmith 等）往往比框架功能更关键
