# 经典 Agent 架构分类：反应式 / 模型式 / 目标驱动 / 效用驱动 / 学习型

## 简单介绍

Agent 架构是指 Agent 内部组件的组织结构及其交互方式。经典 AI 理论将 Agent 架构分为五种基本类型：**反应式（Reactive）**、**模型式（Model-based）**、**目标驱动（Goal-driven）**、**效用驱动（Utility-driven）** 和 **学习型（Learning）**。从简单到复杂逐级递进，每一层在上一层基础上增加新的能力模块。理解这些架构有助于在实际工程中根据场景选择合适的设计层次。

## 基本原理

### 架构演进层次

```
学习型（增加学习模块）         ← 最复杂，能力最强
   ↑
效用驱动（增加效用评估）
   ↑
目标驱动（增加目标状态）
   ↑
模型式（增加内部模型）
   ↑
反应式（最简单的感知-行动映射）  ← 最简单，响应最快
```

## 五种架构详解

### 1. 反应式架构（Reactive Architecture）

**核心机制**：感知 → 直接行动（无内部状态）

```
传感器输入 ─→ 条件-行动规则 ─→ 执行器输出
```

- **特点**：不维护内部世界模型，没有记忆，没有推理
- **优点**：实现简单、响应极快、鲁棒性强
- **缺点**：缺乏灵活性，无法处理需要记忆的复杂任务
- **典型代表**：Brooks 的 Subsumption Architecture（机器人控制）
- **LLM 类比**：简单的单次 Prompt → 回复（无上下文跟踪）

### 2. 模型式架构（Model-based Architecture）

**核心机制**：在反应式基础上增加内部世界模型，用于跟踪环境状态

```
传感器 ─→ 世界模型更新 ─→ 决策 ─→ 行动
               ↑                    │
               └── 状态估计 ←────────┘
```

- **特点**：维护内部模型来补偿感知不足，有"记忆"
- **优点**：能处理部分可观测环境，决策更明智
- **缺点**：模型可能不准确，维护成本增加
- **典型代表**：Dyna-Q 架构（模型 + 规划）
- **LLM 类比**：带上下文管理的对话 Agent

### 3. 目标驱动架构（Goal-driven Architecture）

**核心机制**：在模型式基础上增加目标表示和规划能力

```
传感器 ─→ 模型更新 ─→ 目标评估（当前状态vs目标） ─→ 规划 ─→ 行动
                        ↑                               │
                        └── 差距分析 ←────────────────────┘
```

- **特点**：Agent 不仅有"世界怎么了"的认知，还有"要达成什么"的目标
- **优点**：能自主规划、处理需要长期规划的任务
- **缺点**：目标冲突时难以处理；目标表述太过刚性
- **典型代表**：STRIPS 规划器、HTN 规划
- **LLM 类比**：Plan-and-Execute Agent

### 4. 效用驱动架构（Utility-driven Architecture）

**核心机制**：在目标驱动基础上增加效用函数，量化评估不同状态的"好坏"

```
传感器 ─→ 模型更新 ─→ 效用评估 ─→ 期望效用最大化 ─→ 行动
                           ↑
                ┌── 效用函数（对不同状态的评分）
```

- **特点**：不止判断"是否达到目标"，而是评估"多好地达到目标"
- **优点**：能处理多个冲突目标的权衡；在不确定性下做最优决策
- **缺点**：效用函数设计困难；计算成本高
- **典型代表**：MDP 求解器、理性 Agent
- **LLM 类比**：带多目标权衡的复杂 Agent

### 5. 学习型架构（Learning Architecture）

**核心机制**：在效用驱动基础上增加学习模块，从经验中改进性能

```
传感器 ─→ 学习模块 ─→ 知识库/策略 ─→ 决策 ─→ 行动
            ↑                                    │
            └──────── 经验反馈 ←───────────────────┘
```

- **特点**：能随着时间改进行为，适应环境变化
- **优点**：持续改进、适应新环境
- **缺点**：学习过程不稳定；需要大量数据和精心设计的奖赏机制
- **典型代表**：强化学习 Agent（DQN、PPO）
- **LLM 类比**：Reflexion Agent（从错误中学习）

## 对比总结

| 特性 | 反应式 | 模型式 | 目标驱动 | 效用驱动 | 学习型 |
|------|-------|-------|---------|---------|-------|
| 世界模型 | 无 | 有 | 有 | 有 | 有 |
| 记忆 | 无 | 短期 | 短期 | 短期 | 长期 |
| 目标表示 | 无 | 无 | 有 | 有 | 有 |
| 效用评估 | 无 | 无 | 无 | 有 | 有 |
| 学习能力 | 无 | 无 | 无 | 无 | 有 |
| 计算复杂度 | 低 | 低-中 | 中 | 中-高 | 高 |
| 响应速度 | 快 | 中 | 中-慢 | 慢 | 慢-中 |
| 灵活性 | 低 | 中 | 中-高 | 高 | 很高 |

## 在 LLM Agent 中的对应

有趣的是，**LLM Agent 可以覆盖从反应式到效用驱动的所有层次**，因为 LLM 本身通过预训练已经内化了大量世界知识、推理能力和价值判断：

| LLM Agent 类型 | 对应架构 | 说明 |
|---------------|---------|------|
| Simple I/O Agent | 反应式 | 单次输入输出，无状态 |
| Chain-of-Thought Agent | 模型式 | 维护推理链状态 |
| ReAct Agent | 模型式+目标驱动 | 推理轨迹+行动目标 |
| Plan-and-Execute | 目标驱动 | 明确的规划模块 |
| Reflexion Agent | 学习型 | 自反思+经验积累 |
| Multi-step Agent | 效用驱动 | 评估不同行动路径的期望效用 |

## 核心矛盾

1. **复杂性 vs 响应速度**：越复杂的架构决策质量越高，但响应越慢
2. **灵活性 vs 可预测性**：学习型和效用驱动架构适应性强，但行为更难预测
3. **模型准确性 vs 维护成本**：世界模型越精确，维护成本越高
4. **学习效率 vs 稳定性**：学习能力带来改进潜力，但也可能导致灾难性遗忘或不稳定

## 工程选择指南

| 场景 | 推荐架构 | 理由 |
|------|---------|------|
| 简单固定任务 | 反应式 | 快、省、可靠 |
| 有状态对话 | 模型式 | 需要上下文跟踪 |
| 复杂任务执行 | 目标驱动 | 自主规划执行 |
| 资源优化决策 | 效用驱动 | 多目标权衡 |
| 持续改进系统 | 学习型 | 自适应优化 |
| 大部分 LLM 场景 | 模型式+目标驱动 | 平衡复杂度与能力 |

## 工程优化方向

1. **分层架构**：不同场景使用不同的"心智层次"，简单任务走反应式快速路径，复杂调用目标驱动
2. **混合策略**：多数情况用模型式，经典场景（已学习过）走反应式捷径
3. **架构自适应**：根据任务复杂度动态选择推理深度（类似 System 1 / System 2 思维）
4. **LLM 作为"调谐器"**：LLM 在高层做决策（目标/效用评估），底层执行仍用轻量代码
