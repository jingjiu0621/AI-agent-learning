## 核心思想

1. **Agent 不是新鲜概念**——它的理论根基在 1990 年代就已建立，LLM 让它从理论走向工程
2. **意图驱动 ≠ 状态驱动**——这是 Agent 与传统程序最根本的区别，不只是一个"更智能的 API"
3. **环境决定设计**——Agent 的能力上限不只取决于模型，还取决于它所处的环境类型
4. **BDI 是设计参考不是教条**——借鉴其信念/愿望/意图的分层理念，但不一定严格实现

## 关键工程启示

- Agent 开发者的工作是"引导智能"而非"构建智能"
- 核心循环（感知→思考→行动→反馈）是所有 Agent 系统的基础模式
- 环境分析比模型选择更重要——用错环境假设是 Agent 失败的最常见原因
