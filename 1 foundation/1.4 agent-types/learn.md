# 1.4 agent-types — 主流 Agent 类型与模式

## 本节概述

本节介绍 Agent 的六种主流类型/模式，从最简单的 Simple I/O 到最复杂的 Multi-step Agent。它们不是互斥的分类——而是**能力的递增叠加**，每种新模式在旧模式基础上增加了新的能力维度。

## 内容结构

| 子主题 | 核心能力 | 相比前者的增量 |
|--------|---------|--------------|
| 1.4.1 simple-io | 单次输入输出 | —（基准） |
| 1.4.2 chain-thought | 逐步推理链 | + 中间推理步骤 |
| 1.4.3 react-agent | 推理+工具调用交织 | + 外部工具交互 |
| 1.4.4 reflexion-agent | 自我反思+经验复用 | + 错误回顾与学习 |
| 1.4.5 plan-execute | 先规划后执行 | + 全局规划能力 |
| 1.4.6 multi-step | 多步工具调用+状态跟踪 | + 复杂状态管理 |

## 核心思想

1. **选择模式就是选择复杂度**——每种模式都增加了能力但也增加了 Token 开销和失败概率
2. **ReAct 是当前最实用的"通用模式"**——大多数生产级 Agent 以 ReAct 为基础，叠加其他能力
3. **Reflexion 的"反思"≠"重做"**——反思的价值在于将经验抽象为可复用的知识，而非简单地重试
4. **Plan-and-Execute 适合复杂任务**——但要注意规划本身也是 LLM 调用，规划质量影响一切

## 关键工程启示

- 不是所有 Agent 都需要反思或规划——增加模式前先确认任务复杂度确实需要
- 多步 Agent 的核心挑战不是"推理能力"而是"状态管理"
- 各模式之间可以组合：ReAct + Reflexion 是最常见的生产组合
