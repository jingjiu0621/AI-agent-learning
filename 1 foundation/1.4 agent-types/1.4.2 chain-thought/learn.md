# Chain-of-Thought Agent：逐步推理链

## 简单介绍

Chain-of-Thought（CoT）Agent 在生成最终回复前，先生成中间的推理步骤（"思维链"）。CoT 由 Wei 等人在 2022 年提出，发现通过在 Prompt 中加入"Let's think step by step"就能显著提升 LLM 在复杂推理任务上的准确率。

## 基本原理

### 标准 Prompt vs CoT Prompt

**标准 Prompt**：
```
Q: 小明有 12 个苹果，给了小红 5 个，又买了 3 个，现在有几个？
A: 10 个
```

**CoT Prompt**：
```
Q: 小明有 12 个苹果，给了小红 5 个，又买了 3 个，现在有几个？
A: 开始有 12 个。给了小红 5 个 → 12 - 5 = 7 个。又买了 3 个 → 7 + 3 = 10 个。
所以答案是 10 个。
```

### 两种 CoT 形式

| 形式 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| 零样本 CoT | 直接加"Let's think step by step" | 最简单，无需示例 | 效果不稳定 |
| 少样本 CoT | 提供 2-3 个推理示例 | 更可控 | 需要示例设计 |

## 核心价值

CoT 之所以有效，是因为：
1. **引导中间推理**：LLM 的生成过程被引导为"先推理，后回答"
2. **分解复杂度**：将复杂问题拆解为简单步骤
3. **外显化推理**：推理过程可见，便于调试

## 限制

- 推理链可能包含错误步骤（幻觉）
- 消耗更多 Token（推理步骤的代价）
- 对非常简单的问题来说过度复杂

## 与 ReAct 的关系

CoT = 纯推理（思考但不行动）
ReAct = 推理 + 行动交织（CoT 的扩展——在思考中加入工具调用）

CoT Agent 适合**纯推理任务**（数学、逻辑），ReAct 适合**需要获取外部信息的任务**。

## 工程实现

```
system: "你是一个推理助手。一步一步思考，然后给出答案。"
user: "这里有 7 个红球和 5 个蓝球。拿出了 3 个红球。还剩几个红球？"
assistant (推理链): "最初有 7 个红球。拿出了 3 个红球。7 - 3 = 4。还剩 4 个红球。"
assistant (答案): "4"
```

关键：通过输出 parsed 和结构抽取，将 CoT 的"推理部分"和"答案部分"分开。
