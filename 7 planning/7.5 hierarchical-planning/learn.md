# 7.5 Hierarchical Planning — 分层规划

分层规划是为超长 Horizon、超高复杂度任务设计的规划范式。它通过引入多级抽象，让 Agent 能够系统性地管理从宏观战略到微观执行的完整规划空间。

## 本节概述

分层规划（Hierarchical Planning）的核心思想是"分而治之之再分而治之"——将规划问题分解为不同抽象层次的子问题，每层解决自己层次的问题，将实现细节委托给下一层。这是人类解决复杂问题的标准方法：CEO 制定公司战略，部门经理制定实施计划，员工执行具体任务。

## 核心思想

1. **抽象层次降低复杂度**：每层只关注自己层次的问题，无需同时考虑所有细节
2. **分层规划的核心是接口设计**：层与层之间的"接口"（目标描述、约束传递、结果反馈）决定了分层的有效性
3. **双向信息流**：不仅上层决定下层（自上而下），下层的执行反馈也应影响上层的决策（自下而上）
4. **分层的粒度不是均匀的**：复杂部分需要更多层，简单部分可以更扁平

## 关键工程启示

- 分层规划的优势在任务复杂时体现，在简单任务上是负担
- 典型 LLM Agent 使用"3 层法则"：战略（what）→ 战术（how）→ 执行（do）
- 每层的规划器可以是不同的 LLM 实例，使用不同的 prompt 和 temperature
- 跨层信息传递要保持精炼——原则是"告诉下层需要知道什么，而不告诉下层怎么做的每个细节"
