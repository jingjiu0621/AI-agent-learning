# 7.1 Task Decomposition — 任务分解策略

将复杂任务拆解为可管理、可执行、可验证的子任务是 Agent 规划能力的基石。没有有效的任务分解，Agent 面对复杂目标时就会陷入"不知从何下手"的困境。

## 本节概述

任务分解（Task Decomposition）是 Agent 规划的第一步，也是最关键的一步。它决定了任务的结构、执行顺序和并行度。好的分解能让复杂任务变得清晰可执行，不好的分解会导致遗漏、重复或无法收敛。本节覆盖从层次分解到 DAG 规划、从分解策略到质量评估的完整方法论。

## 核心思想

1. **分解是规划的前提**：LLM 在直接输出完整步骤序列时容易遗漏或出错，结构化分解可以大幅提升可靠性
2. **分解结构决定执行效率**：顺序分解简单但浪费并行机会，DAG 分解复杂但能充分利用资源
3. **分解粒度需要权衡**：太粗导致子任务仍然复杂难以执行，太细导致规划开销过大和管理负担
4. **分解策略应动态选择**：不同任务类型（组合型 vs 流程型 vs 探索型）需要不同的分解策略

## 关键工程启示

- 先用 LLM 生成粗粒度分解，再用规则或二次 LLM 调用细化，比一次性生成细粒度分解更可靠
- 对于已知流程（如"发邮件需要 查联系人→写内容→发送"），预定义的分解模板比 LLM 生成更高效
- 分解时保留"上下文依赖"信息，便于后续步骤引用前置步骤的输出
- 设置最大分解深度（通常 3-5 层）防止过度分解
