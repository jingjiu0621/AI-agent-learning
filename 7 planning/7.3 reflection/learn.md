# 7.3 Reflection — 反思机制与自我改进

反思是 Agent 从"执行者"升级为"学习者"的关键能力。它让 Agent 不仅能做事，还能回顾自己做过的事、分析哪里做得好哪里需要改进、并将经验用于未来的任务。

## 本节概述

反思（Reflection）是 Agent 在完成一个任务或一个步骤后，对自身行为和结果进行回顾、评估和改进的能力。它类似于人类的"事后复盘"——比赛结束后分析自己的走法，找出失误，下次改进。反思是 Agent 持续自我进化的核心驱动力，也是 Reflexion 论文的核心贡献。

## 核心思想

1. **反思是第二次机会**：第一次执行可能失败，但反思能让 Agent 从失败中学习，第二次做得更好
2. **外部视角更客观**：自我反思受限于 Agent 自身的认知偏差，外部 Critic Agent 能提供更客观的评价
3. **时机比深度更重要**：不是每次做错都需要深度反思——简单错误简单纠正，重大失败才深度复盘
4. **反思结果是知识资产**：反思得到的经验应该被存储和复用，而不是用完即弃
5. **反思是一种元认知**：Agent 在思考"自己的思考过程"，这是通向更高级智能的关键能力

## 关键工程启示

- 反思的 prompt 要引导 Agent 关注"为什么失败"而非"失败了什么"
- 外部 Critic 和 Self-Reflection 可以互补：先自省，再请外部评审
- 反思结果需要结构化存储（错误模式→改进策略），方便后续检索和复用
- 过度反思是真实问题——设定反思的最大次数和 token 预算
