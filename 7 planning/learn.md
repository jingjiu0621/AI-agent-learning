# 7 planning — 思考、规划与推理

Agent 的"思考"能力是其区别于简单工具调用的核心特征。规划模块赋予 Agent 将模糊目标转化为可执行步骤、在不确定性中做出决策、从错误中反思改进的能力。

## 本节概述

规划是 Agent 智能的制高点。如果说记忆系统让 Agent"记得住"，工具调用让 Agent"做得到"，那么规划机制决定了 Agent"想得清"——它能将复杂目标分解为可管理步骤，在多个行动路径中做出选择，遇到失败时自我修正，最终可靠地完成任务。本模块深入覆盖从经典 ReAct 模式到分层规划、从任务分解到不确定性处理的完整规划技术栈。

## 核心思想

1. **分解是规划的起点**：任何复杂任务的第一步都是将其拆解为可管理的子任务，分解的质量直接决定了后续执行的成功率。
2. **推理与行动交织**：ReAct 模式证明了"思考-行动-观察"循环比纯推理或纯行动更有效，Reasoning 和 Acting 是相互增强的。
3. **反思构成学习闭环**：Agent 不仅要执行，还要能从执行结果中反思错误、总结经验，形成持续改进的闭环。
4. **规划不是一次性的**：Plan-and-Execute 强调动态调整——计划在执行过程中需要根据实际情况不断修正。
5. **层级抽象管理复杂度**：不同粒度的规划（战略→战术→执行）对应不同的抽象层次，分层可以有效控制规划空间的规模。
6. **不确定性必须被量化**：Agent 需要在不确定时知道自己的不确定程度，并据此做出合理的回退决策。

## 关键工程启示

- 规划器的设计要权衡"规划开销 vs 执行收益"——过度规划比不规划更糟糕
- ReAct 是最实用的基线模式，绝大多数生产系统从它开始
- 反思机制需要精心设计触发条件，避免过度反思导致性能下降
- Plan-and-Execute 适合有明确子步骤依赖关系的任务
- 分层规划是处理超长 horizon 任务的主要手段
- 不确定性处理是人机协作的关键接口点
