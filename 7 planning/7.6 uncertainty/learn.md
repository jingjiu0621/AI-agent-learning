# 7.6 Uncertainty — 不确定性处理与回退

不确定性是 Agent 面对的真实世界常态。如何量化不确定性、在不确定时做出合理决策、以及在错误发生时优雅恢复，是高级 Agent 系统与玩具系统的关键分水岭。

## 本节概述

不确定性处理（Uncertainty Handling）是 Agent 规划能力的"安全网"。即使任务分解、ReAct、反思都做得很好，Agent 仍然会面对"不确定哪个答案正确"、"不确定该选哪个工具"、"不确定任务目标是否理解正确"等场景。处理这些不确定性的能力，决定了 Agent 是从"不确定但勉强回答"升级为"不确定但会主动确认"。

## 核心思想

1. **承认不确定比假装确定更好**：Agent 在不确定时承认并寻求帮助，比给出错误答案更值得信赖
2. **置信度需要量化**："我有点不确定"不够——需要数字化的置信度来指导回退策略的选择
3. **回退不是失败**：优雅的回退（降级、替代、求助）是 Agent 成熟度的标志
4. **不确定性是可以管理的**：通过系统化方法（验证、冗余、交叉检查），大部分不确定性可以被降低到可接受水平

## 关键工程启示

- 设置置信度阈值——低于阈值时主动询问用户或使用 Fallback
- 不确定性来源的追踪比不确定性本身更重要——知道"为什么不确定"比"不确定程度多高"更有用
- 回退策略应该分层：先尝试自动恢复，再降级，最后请求人工
- 概率规划在企业级场景中价值显著（可以量化风险和预期收益）
