# 7.4 Plan-and-Execute — 先规划后执行模式

Plan-and-Execute 将"规划"和"执行"分离为两个明确的阶段——先制定完整的行动计划，再按计划逐步执行。这种模式适合需要全局最优、步骤间依赖复杂的任务。

## 本节概述

与 ReAct 的"边想边做"不同，Plan-and-Execute 采取"先想再做"的策略。规划阶段，Agent 分析任务、制定详细步骤计划；执行阶段，Agent 按计划逐步执行，必要时调整。这种分离拆解了"想"和"做"的耦合，让每个阶段更聚焦。

## 核心思想

1. **全局规划优于局部贪婪**：先通盘考虑能做出更好的决策，避免 ReAct 的"短视"问题
2. **规划与执行解耦**：规划器不受执行时的实时压力影响，可以更深入思考
3. **计划是活的不是死的**：Plan-and-Execute 不是 Plan-then-Do——执行中的反馈应该反哺计划的调整
4. **执行器可以更轻量**：规划阶段承担了大部分推理负担，执行阶段可以更机械地按计划操作

## 关键工程启示

- 规划阶段消耗的 token 在计划复杂任务时通常能在执行阶段数倍地省回来
- 计划应该包含"验证点"——在关键步骤后检查是否按预期推进
- 计划要明确标注每个步骤的输入依赖和预期输出，方便执行时做前后一致性检查
