# 5.1.3 action-selection — 行动选择：贪婪 / 加权 / MCTS 策略

## 简单介绍

Action Selection 是 Agent 主循环中决定"下一步做什么"的核心机制。当 LLM 生成了多个候选行动或行动路径时，需要一种策略来选择执行哪一个。选择策略直接影响 Agent 的效率、探索能力和最终质量。

## 基本原理

在 Agent 循环中，LLM 通常直接输出下一步行动（隐式选择）。但更高级的 Agent 会显式生成多个候选行动，然后通过选择策略从中挑选最优解。

### 三种主要策略

```
贪婪选择（Greedy）：
  候选: [查天气, 查日历, 发邮件]
  评分: [0.9, 0.3, 0.5]
  选择: 查天气（最高分）

加权采样（Weighted）：
  候选: [查天气, 查日历, 发邮件]
  概率: [0.53, 0.18, 0.29]
  随机采样 → 可能选非最高分

MCTS（蒙特卡洛树搜索）：
  1. 选择（Selection）：根据 UCB 值选择节点
  2. 扩展（Expansion）：添加新子节点
  3. 模拟（Simulation）：推演到终局
  4. 回传（Backpropagation）：更新节点评分
```

## 背景与演进

| 时期 | 策略 | 特点 |
|------|------|------|
| 早期 Agent | LLM 直接选（隐式） | 简单但不可控 |
| ReAct | 无显式选择，LLM 直接输出 | 零额外开销 |
| Tree of Thoughts | BFS/DFS 多路径搜索 | 高质量但高成本 |
| 生产 Agent | 混合策略：简单任务贪婪，复杂任务 MCTS | 平衡质量与成本 |

## 核心矛盾

**探索与利用（Exploration vs Exploitation）**：
- 贪婪策略只利用已知最优，可能错过更好的路径
- MCTS 探索充分但计算成本爆炸
- Agent 需要在探索新路径和利用已知路径之间平衡

## 主流优化方向

1. **自适应选择**：根据任务类型动态切换选择策略
2. **分层选择**：高层次（战略层）用搜索，低层次（执行层）用贪婪
3. **价值网络辅助**：训练轻量模型预测行动价值，代替 MCTS 完整模拟
4. **束搜索（Beam Search）**：维护 top-K 路径，而非仅选最优一条

## 实现挑战

1. **评分函数设计**：如何给候选行动打分是最难的——既要考虑当前收益，也要考虑未来潜力
2. **搜索空间爆炸**：MCTS 在多工具场景下搜索空间巨大
3. **延迟代价**：每次选择都增加一次 LLM 调用，延迟线性增长
4. **随机性控制**：加权采样的随机种子和温度参数需要精细调节

## 能力边界

- 贪婪选择无法处理需要"试错"的任务
- MCTS 在即时响应场景中不实用（延迟太高）
- 行动选择的质量最终受限于 LLM 对行动后果的预测能力

## 相关概念区别

- **vs CoT（3.1.1）**：CoT 生成推理链但不做多路径选择
- **vs ToT（3.1.2）**：ToT 是多路径搜索，且包含显式的行动选择策略
- **vs 规划（7.4）**：规划是在行动前全局编排，行动选择是每步的局部决策

## 核心优势

显式的行动选择机制让 Agent 在不确定环境中 **不盲目行动**，而是通过策略性地权衡候选方案来提高成功率。

## 工程优化

1. 简单任务始终用贪婪（零开销）
2. 复杂推理任务用束搜索（Beam=3），替代完整 MCTS
3. 缓存行动评分结果避免重复计算
4. 设置探索预算上限，防止搜索过度

## 场景判断

适合使用高级选择策略（非贪婪）的场景：
- 任务有多种可能的成功路径
- 错误决策的代价很高
- 环境信息不完整，需要探索
