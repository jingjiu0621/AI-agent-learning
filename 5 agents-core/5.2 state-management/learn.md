# 5.2 state-management — Agent 状态管理

## 概念定义

Agent 状态管理是 Agent 在执行过程中维护和更新自身状态的能力。状态包括会话上下文、任务进度、中间结果、工具调用记录等。没有状态管理，Agent 就是"失忆"的——无法在多轮交互中保持一致性。

## 核心角色

| 角色 | 说明 | 实现层面 |
|------|------|---------|
| 状态定义 | 明确哪些数据属于状态 | 数据结构设计 |
| 状态机 | 定义状态间的合法转换 | 状态转移图 |
| 持久化层 | 将状态保存到外部存储 | Redis / DB / 文件 |
| 恢复机制 | 从故障中恢复状态 | 检查点 + 重放 |

## 为什么需要状态管理

- **无状态的 LLM**：LLM 本身是无状态的，每次调用都是独立请求
- **多轮依赖**：当前轮的决策依赖前一轮的执行结果
- **故障恢复**：进程崩溃后需要能恢复执行状态

## 设计范式演进

| 时代 | 方案 | 问题 |
|------|------|------|
| 早期 | 全量消息历史做状态 | Token 爆炸 |
| 框架化 | 结构化 State 对象 | 版本兼容性 |
| 现代 | 不可变事件溯源 | 存储、查询复杂度 |
