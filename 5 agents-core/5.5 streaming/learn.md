# 5.5 streaming — 流式输出与中间状态

## 概念定义

流式输出（Streaming）是 Agent 将输出逐步推送给客户端而非一次性返回的技术。在 Agent 场景中，流式输出不仅包括 Token 流，还包括中间状态（思考过程、工具调用进度）的实时推送。

## 核心挑战

| 挑战 | 描述 | 影响 |
|------|------|------|
| 延迟感知 | 用户等待时感知时间变长 | 用户体验差 |
| 进度透明 | Agent 在做啥用户不知道 | 不确定感 |
| 部分输出 | 生成未完成时输出有效 | 技术挑战 |
| 中断控制 | 用户可以停止正在执行的 Agent | 实现复杂 |

## 传输协议

| 协议 | 适用场景 | 特点 |
|------|---------|------|
| SSE | 服务器→客户端单向流 | 简单、浏览器原生支持 |
| WebSocket | 双向实时通信 | 支持双向、稍复杂 |
| Chunked Transfer | HTTP 分块传输 | 标准 HTTP、兼容性好 |
