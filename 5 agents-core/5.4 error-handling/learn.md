# 5.4 error-handling — 错误处理与恢复策略

## 概念定义

Agent 错误处理是在 Agent 执行过程中检测、分类和恢复错误的能力。Agent 的错误来源比传统程序更复杂——除了代码本身的 bug，还有 LLM 推理错误、工具调用失败、环境变化等。

## 错误来源分类

| 错误来源 | 示例 | 频率 | 处理难度 |
|---------|------|------|---------|
| LLM 推理错误 | 幻觉、逻辑错误、指令理解偏差 | 中-高 | 高 |
| 工具执行错误 | API 超时、权限拒绝、返回值异常 | 中 | 中 |
| Agent 逻辑错误 | 循环陷阱、状态不一致、选择错误 | 低-中 | 高 |
| 环境错误 | 网络中断、配置错误、依赖缺失 | 低 | 低 |

## 核心策略

| 策略 | 思想 | 成本 | 适用 |
|------|------|------|------|
| 优雅降级 | 部分功能失效时继续运行 | 低 | 非关键功能 |
| 重试 | 失败后重试（带退避） | 低-中 | 临时故障 |
| 替代路径 | 换一种方式完成任务 | 中 | 多种解决方案 |
| 人工升级 | 无法处理时交给人类 | 高 | 关键决策 |
| 熔断 | 检测到系统异常时停止尝试 | 低 | 级联故障防护 |
