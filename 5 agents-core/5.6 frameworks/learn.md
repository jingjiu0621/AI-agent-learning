# 5.6 frameworks — 框架源码深度分析

## 概念定义

框架源码分析是深入理解主流 Agent 框架（LangGraph、CrewAI、AutoGen）的核心机制，并通过手写实现加深对 Agent 架构的理解。这是从"框架使用者"到"框架理解者"的关键一步。

## 核心分析维度

| 维度 | 说明 | 学习价值 |
|------|------|---------|
| 执行模型 | 图 / 管道 / 对话 | 理解框架的设计哲学 |
| 状态管理 | 状态定义、传递、持久化 | 学习状态管理的最佳实践 |
| 流式机制 | 如何支持流式输出 | 工程实现技巧 |
| 错误处理 | 框架层的错误恢复 | 生产级设计模式 |
| 扩展性 | 如何自定义和扩展 | 设计模式的应用 |

## 框架对比总览

| 框架 | 核心模型 | 状态管理 | 适用场景 |
|------|---------|---------|---------|
| LangGraph | 有向图（StateGraph） | TypedDict State + Checkpointer | 复杂工作流编排 |
| CrewAI | 角色分工（Agent/Task/Crew） | 隐式消息传递 | 多角色协作 |
| AutoGen | 对话驱动（Agent Chat） | 对话历史 | 多 Agent 对话 |
| 手写实现 | 自定义循环 | 自定义 | 学习/轻量场景 |
