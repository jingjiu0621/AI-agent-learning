# 5.3 message-history — 消息历史管理与压缩

## 概念定义

消息历史管理是 Agent 维护和优化 LLM 对话上下文的技术。由于 LLM 的上下文窗口有限，且 Token 消耗直接关联成本和延迟，如何高效管理不断增长的消息历史是生产级 Agent 的核心工程问题。

## 核心挑战

| 挑战 | 描述 | 影响 |
|------|------|------|
| Token 预算 | 上下文窗口有限（8K-200K） | 长任务无法完成 |
| 成本控制 | Token 越多成本越高 | 经济不可持续 |
| 信息保留 | 关键信息不能丢失 | 任务质量下降 |
| 近因偏差 | LLM 倾向关注最新内容 | 早期重要信息被忽略 |

## 主要策略

| 策略 | 核心思想 | 适用场景 |
|------|---------|---------|
| 滑动窗口 | 只保留最近 N 轮消息 | 简单对话 |
| 自动摘要 | 将旧消息压缩为摘要 | 长对话 |
| 分层架构 | 核心+压缩+长期三层次 | 复杂 Agent |
| 重要性裁剪 | 按重要性丢弃消息 | Token 敏感场景 |
