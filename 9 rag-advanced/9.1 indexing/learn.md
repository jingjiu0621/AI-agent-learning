# 9.1 indexing — 索引策略

索引是 RAG 系统的数据基础设施。没有高质量的索引，再先进的检索算法也无法找到正确的信息。索引策略决定了"什么是可以被检索到的"——它是对原始文档的重新组织、标注和结构化过程。

## 核心设计决策

### 分块粒度光谱

```
精细分块                          粗粒度分块
◄──────────────────────────────────────────────────►
  128 tokens    512 tokens    1024 tokens   2048+ tokens
  高精度          中等精度        中等精度         低精度
  低上下文完整     中等完整        较好完整         高上下文完整
  高检索召回      适中召回        适中召回         低检索召回
```

**没有绝对正确的分块大小——取决于你的检索场景：**
- 事实性 QA：小分块（128-256 tokens）→ 精准定位答案
- 摘要性任务：大分块（512-1024 tokens）→ 需要完整上下文
- 代码检索：函数/类级别 → 结构完整而非 token 数量

### 索引层级关系

```
                      ┌─────────────────────────┐
                      │    摘要索引 (粗粒度)       │
                      │  每个文档/章节 → 摘要      │
                      │  快速初筛，找到相关文档      │
                      └──────────┬──────────────┘
                                 │ 命中
                      ┌──────────▼──────────────┐
                      │    全文索引 (中粒度)       │
                      │  分块 → 向量 + BM25      │
                      │  精确定位相关段落          │
                      └──────────┬──────────────┘
                                 │ 需要细节
                      ┌──────────▼──────────────┐
                      │    详细索引 (细粒度)       │
                      │  句子/关键信息点           │
                      │  精确提取答案              │
                      └─────────────────────────┘
```

## 关键权衡

| 决策维度 | 倾向 A | 倾向 B | 选择依据 |
|---------|--------|--------|---------|
| 分块大小 | 小(128-256) | 大(1024+) | 任务类型：事实性 vs 摘要性 |
| 分块重叠 | 有重叠(10-20%) | 无重叠 | 信息完整性 vs 存储效率 |
| 索引层级 | 多级索引 | 单级索引 | 知识库规模：大 → 多级 |
| 元数据粒度 | 精细标注 | 少量标注 | 过滤需求：频繁过滤 → 精细 |
| 更新策略 | 增量更新 | 全量重建 | 数据变化频率：高频 → 增量 |
| 存储介质 | 向量库+全文 | 纯向量 | 检索场景：精确匹配需要全文 |
