# 10.5 Visualization — Agent 行为可视化

Visualization（可视化）是 Agent 可观测性的"呈现层"。它将日志、追踪、监控中的结构化数据转化为直观的图形表示，使开发者可以"一目了然"地理解 Agent 的行为模式。

## 为什么需要可视化？

Agent 系统的高维、非线性、非确定性的行为模式，使得纯文本的日志和数字指标难以理解：

| 数据类型 | 纯文本/数字 | 可视化 |
|---------|-----------|--------|
| Agent 推理路径 | 8 页 Thought 文本 | 一张思维图谱 |
| 工具调用 DAG | 50 行日志 | 一张调用拓扑图 |
| 多 Agent 交互 | 200 条消息记录 | 一张泳道图 |
| 延迟分布 | P50/P95/P99 三个数字 | 一张延迟瀑布图 |
| 记忆检索 | 100 行检索记录 | 一张关联强度图 |

## 核心设计原则

1. **"一眼"传达核心信息**——好的可视化不需要读标题和说明就能理解 Agent 的当前状态
2. **交互式探索**——静态图表是不够的，开发者需要可以展开/折叠/过滤/搜索的可视化
3. **时间维度贯穿**——Agent 的执行是时间序列，可视化需要将时间维度直观展示（时间线、动画、水流图）
4. **适度抽象**——不需要展示所有细节，关键是在"可读性"和"信息量"之间做取舍
5. **与日志/追踪双向链接**——在可视化中点击一个节点，跳转到对应的日志或 Trace
