# 10.5.3 Memory Visualization — 记忆检索路径

## 简单介绍

Memory Visualization（记忆可视化）展示 Agent 的记忆检索过程——查询是什么、检索到了哪些记忆、哪些被使用了、哪些被忽略了、检索结果的关联强度。这对于调试 Agent 的"知识使用"行为至关重要：**Agent 是不是基于正确的记忆在做决策？**

## 基本原理

记忆检索过程可视化的核心组件：

```python
@dataclass
class MemoryRetrievalEvent:
    query: str                          # 检索查询
    query_embedding_hash: str           # 查询向量的哈希
    retrieval_strategy: str             # semantic / keyword / hybrid
    
    candidates: list[MemoryCandidate]   # 检索到的候选项
    top_k: int                          # 返回的 top_k
    threshold: float                    # 相似度阈值
    
    # 检索结果
    results: list[MemoryResult]         # 最终返回的结果
    latency_ms: float
    
@dataclass
class MemoryResult:
    memory_id: str
    content_summary: str                # 记忆内容摘要
    similarity_score: float             # 相似度分数
    was_used: bool                      # 是否被 Agent 实际使用
    rank: int                           # 排序位置
```

记忆检索路径的可视化：

```
查询: "北京今天天气怎么样？"
    │
    ▼
[语义检索] ────────────────────────────── [相关性阈值: 0.75]
    │
    │  候选结果 (similarity):
    │
    ├── 1. "北京的天气是..."  0.92  ✅ 被使用
    │      来源: 天气API缓存 (10分钟前)
    │
    ├── 2. "上海的温度是..."  0.78  ✅ 被使用
    │      来源: 用户之前问过上海天气
    │
    ├── 3. "北京的景点有..."  0.65  ❌ 低于阈值
    │      来源: 旅游指南记忆
    │
    └── 4. "北京的人口..."    0.52  ❌ 低于阈值
           来源: 百科知识
    │
    ▼
[最终使用的记忆] → Agent 参考了记忆 #1 和 #2 生成回答
```

## 背景

记忆系统是 Agent 的"知识底座"（Module 6），但"记忆中有什么"和"Agent 实际用了什么"之间的差距往往很大。记忆可视化填补了这个"检索黑盒"——让开发者看到 Agent 怎么查的、查到了什么、用了什么。

## 之前的做法与结果

| 做法 | 描述 | 结果 |
|------|------|------|
| 不展示检索过程 | 只看 Agent 最终输出 | 无法知道 Agent 是否使用了正确的记忆 |
| 文本日志 | 输出 "retrieved 5 memories" | 只告诉数量，不告诉内容和相关性 |
| 手动检查数据库 | 去向量数据库中查看记忆 | 没有与具体查询的关联，无法判断检索质量 |
| 只看命中率 | 监控检索的命中/未命中 | 命中率高不代表用的好 |

## 当前主流优化方向

1. **语义空间可视化**——将记忆的向量映射到 2D/3D 空间（使用 t-SNE/UMAP 降维），展示查询向量在语义空间中的位置以及检索到的记忆分布
2. **记忆使用链路图**——展示"用户输入 → 记忆检索 → Thought 引用 → Action 执行 → 新的记忆存储"的完整链路
3. **检索质量热力图**——用颜色标识每次检索的"质量"：检索到的记忆是否相关、是否被使用、是否对最终输出有贡献
4. **时间轴记忆视图**——将记忆按时间排列在时间轴上，标记"记忆创建时刻"和"记忆被引用时刻"，直观展示记忆的"生命周期"

## 实现的最大挑战

1. **语义空间的可解释性**——降维后的向量空间虽然可视但难以解释，"为什么这两段记忆在语义上接近"很难从 2D 坐标中理解
2. **大规模记忆的展示**——Agent 可能有数百万条记忆，无法全部展示，需要有效的摘要和采样策略
3. **跨模态记忆可视化**——多模态记忆（文本+图像+结构化数据）需要在同一可视化中展示不同类型的数据
4. **检索效果的量化评价**——"这个检索好吗？"缺少直观的度量标准，可视化只能展示"发生了什么"而非"好不好"

## 能力边界

**能做什么：**
- 展示 Agent 检索记忆的完整过程（查询→候选→结果→使用）
- 帮助调试"Agent 用了错误的记忆"或"Agent 忽略了正确的记忆"等问题
- 直观评估检索质量（检索到的结果与查询的相关性）
- 发现记忆系统的配置问题（top_k 不合适、阈值不对、embedding 不一致）

**不能做什么：**
- 不能直接展示"记忆中的全部内容"——只能展示检索到的结果
- 不能自动判断"检索是否充分"——Agent 可能检索到了合适的记忆但没有正确使用，这是推理问题不是检索问题
- 不能替代 Embedding 模型的评估——对于检索不到的情况，需要区分是"没有相关记忆"还是"Embedding 模型无法捕捉语义相似度"

## 最终工程优化

1. **检索摘要卡片**——将每次检索的关键信息浓缩为一张信息卡片（查询、结果数量、使用数量、平均相似度），可以快速扫描 Agent 的"记忆使用习惯"
2. **记忆命中率时间线**——按时间绘制"检索命中率"曲线，发现 Agent 在"什么时间段"或"处理什么类型任务时"记忆命中率低
3. **记忆使用占比分析**——饼图展示"本次任务中 Agent 使用了多少记忆 vs 多少工具结果 vs 多少自身知识"，发现 Agent 是否过度依赖某一信息源
4. **AB 对比模式**——同一查询的两种记忆配置（不同 Embedding 模型、不同 top_k、不同检索策略）的检索结果并列对比，直观评估优化效果
