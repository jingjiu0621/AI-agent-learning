# 8 multi-agent — 多智能体协作与组织

多智能体系统是 Agent 技术的最高级组织形式。当单个 Agent 的能力达到瓶颈时，多个 Agent 通过通信、协调、委派和辩论，能够解决远超单个 Agent 能力范围的复杂问题。

## 本节概述

如果说单个 Agent 是"一个人的公司"，那么多智能体系统就是"一个完整的组织"。多 Agent 不是简单地把多个 LLM 拼在一起，而是需要设计通信协议、协调机制、任务分配策略、辩论协商规则以及群体组织结构。本模块深入覆盖从 Agent 间通信到群体组织模式的完整多智能体技术栈。

## 核心思想

1. **通信是协作的基础**：没有高效的通信协议，多 Agent 系统就无法存在。消息格式、同步/异步、路由机制决定了系统的信息流通效率。

2. **协调防范冲突**：多个 Agent 独立行动必然产生资源竞争和目标冲突。协调机制（中心化/去中心化、投票、共识）是避免"三个和尚没水喝"的关键。

3. **委派是对能力的组织**：任务分配不是简单的"谁有空给谁"，而是基于能力、负载、优先级和上下文的智能路由。好的委派策略让系统整体效率远高于个体之和。

4. **辩论产生更好的决策**：多 Agent 辩论不是争吵，而是通过"主张→证据→反驳→评审"的结构化过程，让不同视角的观点碰撞，最终产生更鲁棒的结论。

5. **群体模式决定系统行为**：Manager-Worker、议会、市场、流水线、网状——不同的组织模式适用于不同的问题域。选对模式比调优参数更重要。

6. **多 Agent 引入新维度复杂性**：通信开销、共识延迟、冲突解决、一致性问题——这些分布式系统的经典难题在多 Agent 系统中以新的形式出现。

## 关键工程启示

- 多 Agent 系统不是越多越好——每增加一个 Agent 都引入通信开销和协调成本
- 通信协议设计要优先考虑容错性：消息可能丢失、延迟、乱序
- 中心化协调器是瓶颈也是单点故障，去中心化则面临一致性挑战
- 任务分配时要考虑 Agent 的"能力边界"——不是所有 Agent 都能做所有事
- 辩论机制需要"评审 Agent"来裁决，否则可能陷入无限循环
- 群体模式可以混合使用（如管理器+流水线混合），灵活组合比单一模式更强大
- 生产级多 Agent 系统必须考虑：消息持久化、重试、超时、死信队列
