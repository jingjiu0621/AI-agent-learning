# 8.1 communication — Agent 间通信协议

## 概述

多 Agent 系统中的通信协议定义了 Agent 之间如何交换信息、如何理解彼此的消息、以及如何保证消息的可靠传递。通信是协作的基础——没有高效的通信，多 Agent 系统只是一堆各自为政的 LLM 实例。

## 关键设计原则

1. **消息设计先行**：消息格式决定了系统的表达能力、可扩展性和调试难度
2. **同步适合短操作**：请求-响应模式适合快速、确定性的交互
3. **异步适合长流程**：消息队列解耦发送方和接收方，提高系统弹性
4. **广播要谨慎**：广播是 O(n) 的通信复杂度，大规模系统需要选择性分发
5. **结构化降低歧义**：角色标记和消息类型让 Agent 正确理解消息的"言外之意"
6. **开销是硬约束**：通信延迟、带宽、序列化成本在规模扩大时快速增长
